## 问题描述

早间报警触发两类生产故障：

### 1. data 服务容器路径问题

`jbt-data` 容器（Mini）在切换为 `directory=/app` 后，`supervisord.conf` 中的 command 路径（`src.main` 短路径）与新 PYTHONPATH 不兼容，导致 API/Scheduler/Health 三个进程启动异常。

**根因**：`directory` 已改为 `/app`（工作区根目录），但 command 仍使用子路径 `src.main`，导致模块找不到。同时 `configs/settings.yaml` 中 `storage.base_path` 仍为 `~/jbt-data`（宿主机路径）而非容器路径 `/data`，`Dockerfile` 也需对应更新依赖和构建参数。

### 2. sim-trading CTP 守护协程报警噪音

CTP guardian 在非交易时段（夜间、周末）持续触发误报，时段判断逻辑存在缺陷，`shutdown` 事件也触发无效报警。
