## 背景

1. 当前 data 端“情绪指数”已稳定运行，但经只读核查，现有实现仅覆盖 A 股/全市场截面情绪。
2. 用户已明确要求：在当前“情绪指数”能力基础上，增加期货 35 个品种维度。
3. 本事项属于 data 单服务内的新功能扩展，不属于 U0 单点热修，必须走标准建档 → 预审 → 白名单 → Token 流程。

## 已确认事实

1. 当前采集器实现位于 `services/data/src/collectors/sentiment_collector.py`，数据源只有融资融券、北向资金、QVIX、市场活跃度，没有期货 35 品种维度。
2. 当前落盘与上下文出口位于：
   - `services/data/src/scheduler/pipeline.py`
   - `services/data/src/data/sentiment_mapper.py`
   - `services/data/src/api/routes/context_route.py`
3. 当前 `context/sentiment` 只读取 `sentiment/sentiment` 这一全市场情绪桶，不存在按期货品种输出结构。
4. data 服务内部已存在可复用的 35 品种资产：
   - `services/data/src/researcher/config.py` 中的 35 品种列表
   - `services/data/src/researcher/models.py` 中的 `affected_symbols` 与 `sentiment` 字段
   - `services/data/src/main.py` / `services/data/src/researcher_store.py` 中已存在 Mini 本地研究员报告存档与 `/api/v1/researcher/report/latest` 读取能力
5. 仓内还存在 decision 侧 35 品种新闻打标白名单，但本批默认不跨服务改动，不依赖 decision 实现。

## 服务边界

1. 本批仅归属 `services/data/`。
2. 不修改 `services/decision/**`、`services/dashboard/**`、`shared/**`、`integrations/**`。
3. 不修改任一部署文件、真实环境变量、运行态数据目录。

## 拟定实现口径

1. 保留现有全市场情绪桶 `sentiment/sentiment` 不变，避免回归当前看板与既有消费方。
2. 在 data 单服务内部新增“期货 35 品种情绪聚合”产物，优先复用 Mini 已落地的研究员报告/文章分析结果中的：
   - `affected_symbols`
   - `sentiment`
3. 不允许把现有 A 股情绪值简单复制到 35 个期货品种；必须输出真正具备品种维度的聚合结果。
4. 上下文 API 应新增独立出口，避免覆盖现有 `/api/v1/context/sentiment` 语义。

2. 若需读取 `researcher_store.py` 或研究员模型，但仅为复用现有接口/结构，默认先不纳入写入白名单；只有确认必须修改时才补充预审。

## 验收标准

1. 明确区分“全市场情绪”与“期货 35 品种情绪”，两者数据语义不可混淆。
2. 新增的期货情绪数据至少覆盖既定 35 品种，且每条记录能标识具体品种。
3. 现有 `/api/v1/context/sentiment` 行为保持兼容，不因本批回归。
4. 新增上下文/查询出口可稳定返回 35 品种情绪结果。
5. data 服务本地最小测试通过，且情绪相关现有测试不回归。
6. 白名单外零写入。
