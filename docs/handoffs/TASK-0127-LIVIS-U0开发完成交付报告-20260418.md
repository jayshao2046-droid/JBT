## 四、技术架构

### 4.1 组件依赖关系

```
主控脚本 (run_full_pipeline_35_symbols.py)
    │
    ├─→ TqSdkBacktestClient (TqSdk 回测)
    ├─→ EvidenceManager (证据管理)
    ├─→ FeishuStrategyNotifier (飞书通知)
    ├─→ ContractResolver (合约解析)
    ├─→ YAMLSignalExecutor (本地回测)
    ├─→ StrategyParamOptimizer (调优)
    └─→ StrategyEvaluator (评分)
```

### 4.2 数据流

```
1. 策略生成 → llm_generated/{symbol}/{strategy_name}.yaml
2. 调优报告 → llm_generated/{symbol}/{strategy_name}/reports/optimization_report.json
3. 本地回测 → llm_generated/{symbol}/{strategy_name}/reports/local_backtest_report.json
4. TqSdk 回测 → llm_generated/{symbol}/{strategy_name}/reports/tqsdk_backtest_report.json
5. 评分报告 → llm_generated/{symbol}/{strategy_name}/reports/evaluator_report.json
6. 分桶迁移 → llm_ranked/{score_bucket}/{symbol}/{strategy_name}/
7. 飞书通知 → 实时发送
```
