## 二、决策端通知体系部署

### 核心模块

#### 1. 门控通知 (`gate_notifications.py`)
- **L1 拒绝通知**: P0/P1 级别，60秒去重，二次升级机制
- **L2 通过通知**: NOTIFY 级别，记录通过原因
- **L3 执行通知**: P2 级别，记录确认/超时/失败
- **策略调优通知**: P1 级别，记录调优失败原因

#### 2. 每日汇总 (`daily_summary.py`)
- L1/L2/L3 统计（通过率、拒绝率、确认率）
- 品种命中率分析
- 拒绝原因 TOP 5
- 失败通知统计

#### 3. 健康监控 (`health_monitor.py`)
- 5 大组件监控（信号源/门控/调度/执行/通知）
- 15 分钟快照（降级状态告警）
- 每日健康报告（24小时趋势分析）

### 通知分层设计

| 级别 | 触发条件 | 通道 | 去重 |
|------|---------|------|------|
| P0 | L1 拒绝（高风险） | 飞书 + 邮件 | 60秒 |
| P1 | L1 拒绝（中风险）、策略调优失败 | 飞书 + 邮件 | 60秒 |
| P2 | L3 超时/失败 | 飞书 + 邮件 | 60秒 |
| NOTIFY | L2 通过 | 飞书 | 60秒 |
| DAILY | 每日汇总 | 邮件 | - |
| HEALTH | 健康快照/报告 | 飞书 + 邮件 | - |
