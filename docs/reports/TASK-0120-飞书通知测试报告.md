## 问题根因

资讯群的 webhook (`c51f4956-2a5c-4cad-86c2-17684da5e4da`) 配置了**关键词白名单**，任何不包含指定关键词的消息都会被拒绝。

## 环境变量配置问题

检查 `services/data/.env` 发现：
```bash
FEISHU_ALERT_WEBHOOK_URL=...622ca546...     # 报警群 ✅
FEISHU_TRADING_WEBHOOK_URL=...bfe0f163...   # 交易群 ✅
FEISHU_NEWS_WEBHOOK_URL=...c51f4956...      # 资讯群 ❌ (有关键词限制)
```

但 `researcher/notifier.py` 使用的是：
```python
self.feishu_webhook = os.getenv("FEISHU_WEBHOOK_URL", "")  # ← 这个变量不存在！
```

**生产环境问题**：研究员通知根本发不出去，因为环境变量不匹配。

## 解决方案

### 方案 1: 修改代码使用正确的环境变量（推荐）
修改 `researcher/notifier.py`：
```python
# 改为
self.feishu_webhook = os.getenv("FEISHU_NEWS_WEBHOOK_URL", "")
```

但这样仍然会遇到关键词问题，需要配合方案 2 或 3。

### 方案 2: 在飞书机器人设置中移除关键词限制
登录飞书开放平台，找到资讯群对应的机器人，移除 webhook 的关键词白名单配置。

### 方案 3: 创建新的 webhook（最简单）
创建一个新的飞书机器人，不配置关键词限制，将新的 webhook URL 设置为 `FEISHU_NEWS_WEBHOOK_URL`。

### 方案 4: 提供正确的关键词
如果资讯群必须保留关键词限制，请提供正确的关键词，我们在测试和生产代码中使用。
