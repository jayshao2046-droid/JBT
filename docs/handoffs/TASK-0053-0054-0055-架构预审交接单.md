## 关键发现（架构师必读）

通过代码审查，发现以下对预审决策有影响的事实：

### 1. FactorLoader 已有 httpx 客户端模式

`services/decision/src/research/factor_loader.py` 已通过 `httpx` 调用 data service bars API（`_fetch_bars` 方法）。  
→ **TASK-0053（C0-2）不需要新建独立 HTTP 客户端类**，只需在现有 `FactorLoader` 扩展股票 symbol 支持即可，`stock_data_client.py` 可精简或合并。

### 2. Decision 服务无 watchlist API

`services/decision/src/api/routes/` 目录下共有 5 个路由文件（approval/health/model/signal/strategy），  
`strategy.py` 前缀为 `/strategies`，路由有 overview/list/detail/publish/dashboard，**无 watchlist 端点**。  
→ **TASK-0054（CB5）前置依赖缺口**：需先在 decision 服务新增 `GET /strategies/watchlist` 或类似端点，才能让 CB5 的 watchlist_client 有调用目标。  
→ **架构师裁定**：① 作为 TASK-0054 的子范围在同一 Token 内完成？② 还是拆分为独立 TASK-005X？

### 3. Backtest 路由注册在 api/app.py

`services/backtest/src/main.py` 仅作为 uvicorn 入口，所有路由通过 `services/backtest/src/api/app.py` 的 `create_app()` 注册。  
→ **TASK-0055（CG2）白名单已更正**，修改文件为 `api/app.py` 而非 `main.py`。
