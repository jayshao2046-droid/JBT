## 当前问题

**核心问题**：V3优化了入场/出场逻辑，但CodeGenerator生成YAML时market_filter仍是旧值

**实际案例**：
```yaml
# V3优化后的入场条件
signal:
  long_condition: "atr > 0.0008 * close and volume_ratio > 1.3"
  short_condition: "volume_ratio < 0.95"

# 但market_filter未更新
market_filter:
  conditions:
    - "atr > 0.006 * close"      # ❌ 应该是 0.002
    - "volume_ratio > 1.0"        # ❌ 应该是 0.9
```

**影响**：
- 市场过滤的ATR阈值（0.006）比入场条件（0.0008）高7.5倍
- 市场过滤要求volume_ratio>1.0，但出场条件要求<0.95，逻辑互斥
- 导致大部分信号被过滤掉，策略无法正常工作

**方案**：修改 `services/decision/src/research/code_generator.py` 的prompt

**具体修改**：
在CodeGenerator的prompt中添加以下内容：

```python
"""
【重要】市场过滤条件同步规则：

1. 如果策略设计中提到"移除市场过滤"或"不需要市场过滤"：
   - 完全删除market_filter section

2. 如果策略设计中提到"放宽市场过滤"或给出了具体阈值：
   - 必须同步更新market_filter中的conditions
   - 使用策略设计中提到的新阈值

3. 市场过滤的设计原则：
   - market_filter的阈值应该比入场条件更宽松
   - 例如：如果入场条件是 atr > 0.002 * close，
     那么market_filter应该是 atr > 0.001 * close 或者不设置ATR过滤
   - 避免market_filter与出场条件逻辑冲突
   - 例如：如果出场条件是 volume_ratio < 0.95，
     那么market_filter不应该要求 volume_ratio > 1.0

4. 如果策略设计中没有明确提到market_filter：
   - 检查入场条件中的阈值
   - 自动生成更宽松的market_filter（阈值降低50%）
   - 或者不生成market_filter section
"""
```

### 批次 B：优化策略生成逻辑（P1）

**目标**：V3生成的初始策略不要过于保守

**方案**：修改 `services/decision/src/research/strategy_architect.py` 的prompt

**具体修改**：
在StrategyArchitect的prompt中添加以下内容：

```python
"""
【策略设计原则】

1. 初始策略应该适度宽松，避免0交易
   - 目标：回测期间（2年）至少产生10-20笔交易
   - 如果预期交易次数<10，说明条件过严

2. 市场过滤条件应该比入场条件更宽松
   - 市场过滤是全局筛选，应该过滤掉极端异常行情
   - 入场条件是精确信号，可以更严格
   - 错误示例：market_filter要求atr>0.01，入场要求atr>0.005（冲突）
   - 正确示例：market_filter要求atr>0.002，入场要求atr>0.005（合理）

3. 避免三重叠加
   - 不要同时使用：严格的市场过滤 + 严格的入场条件 + 严格的出场条件
   - 建议：市场过滤宽松 + 入场条件适中 + 出场条件灵活

4. 避免逻辑冲突
   - 市场过滤与出场条件不能互斥
   - 错误示例：market_filter要求volume_ratio>1.0，出场要求volume_ratio<0.8
   - 正确示例：market_filter要求volume_ratio>0.5，出场要求volume_ratio<0.8

【参数建议范围】

- ATR阈值：
  * 市场过滤：0.001-0.003 * close（或不设置）
  * 入场条件：0.002-0.008 * close
  * 不要超过0.01 * close

- VolumeRatio：
  * 市场过滤：0.5-0.8（或不设置）
  * 入场条件：1.2-1.8
  * 出场条件：0.6-0.9
  * 不要使用>2.0的极端值

- ADX：
  * 入场条件：15-25
  * 不要超过30

- Bollinger偏移：
  * 使用相对偏移：0.1-0.5倍带宽
  * 不要直接使用上轨/下轨突破（太极端）
"""
```
