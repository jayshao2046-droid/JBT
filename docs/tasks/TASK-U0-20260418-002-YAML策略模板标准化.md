## 3. TqSDK 格式关键差异

### 3.1 格式对比表

| 字段 | 自定义格式 | TqSDK 标准格式 | 说明 |
|-----|-----------|---------------|------|
| **版本信息** | ❌ 无 | `version`, `category` | TqSDK 必需 |
| **因子权重** | ❌ 无 | `factors[].weight` | 每个因子需要权重（0-1） |
| **市场过滤** | `market_filter.conditions` | `market_filter.enabled`, `conditions` | 需要明确开关 |
| **仓位调整** | ❌ 无 | `position_adjustment` | 支持动态调整 |
| **止损止盈** | `stop_loss_yuan: 1000` | `stop_loss.atr_multiplier: 1.5` | 使用 ATR 倍数 |
| **强制平仓** | `time_to_exit_bars: 12` | `force_close_day: "14:55"` | 使用具体时间点 |
| **初始资金** | `initial_capital: 500000` | ❌ 不支持 | 通过 `TqSim()` 设置 |
| **标签** | ❌ 无 | `tags: ["v1", "trend"]` | 支持标签分类 |

### 3.2 TqSDK 不支持的字段

以下字段在 TqSDK 格式中**不支持**，CodeGenerator 不应生成：

❌ `initial_capital`: 初始资金（通过 `TqSim(initial_capital)` 设置）  
❌ `stop_loss_yuan`: 固定金额止损（必须使用 `stop_loss.atr_multiplier`）  
❌ `take_profit_yuan`: 固定金额止盈（必须使用 `take_profit.atr_multiplier`）  
❌ `time_to_exit_bars`: K 线数强制平仓（必须使用 `force_close_day`）  
❌ `consecutive_loss_pause_days`: 连续亏损暂停（TqSDK 不支持）  
❌ `max_position_per_day`: 每日最大开仓次数（TqSDK 不支持）  
❌ `margin_usage_percent`: 保证金使用率（TqSDK 不支持）

### 3.3 TqSDK 必需的字段

以下字段在 TqSDK 格式中**必须包含**：

✅ `version`: 版本号（如 "1.0"）  
✅ `category`: 策略分类（trend_following / mean_reversion / breakout / arbitrage / intraday_momentum / intraday_oscillation / intraday_reversal）  
✅ `factors[].weight`: 因子权重（0-1，总和建议为 1.0）  
✅ `market_filter.enabled`: 市场过滤开关（true/false）  
✅ `position_adjustment`: 动态仓位调整配置  
✅ `transaction_costs`: 交易成本（必填，否则无法回测）  
✅ `stop_loss.atr_multiplier`: 止损 ATR 倍数  
✅ `take_profit.atr_multiplier`: 止盈 ATR 倍数  
✅ `risk.force_close_day`: 日盘强制平仓时间  
✅ `tags`: 标签列表
