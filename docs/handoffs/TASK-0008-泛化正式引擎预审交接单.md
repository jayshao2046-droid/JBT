## 当前技术结论摘要

1. TQSDK 直接接收的会话层参数冻结为：`TqAuth(user_name, password)`、`TqBacktest(start_dt, end_dt)`、`TqSim(init_balance)`、`TargetPosTask(api, symbol, price, offset_priority, min_volume, max_volume, account)`。
2. `tqsdk 3.9.1` 中，`TqSim` 有 `set_commission(symbol, commission)`，没有 `set_slippage`。
3. 当前仓内只有 FC-224 模板被注册，现有结构仍是 `StrategyTemplateRegistry + FixedTemplateStrategy + FC224Strategy` 的专用实现。
4. 当前仓内没有现成的泛化表达式引擎；`market_filter / signal` 虽可从 YAML 解析，但执行逻辑仍硬编码在 `fc_224_strategy.py`。
5. 当前真正闭环的风控只包括 `position_fraction / position_adjustment`、`force_close_day`、`force_close_night`、`no_overnight`；`stop_loss / take_profit / max_drawdown / daily_loss_limit` 等尚未闭环。

## 五大参数与执行口径摘要

1. 参数大类冻结为：资金参数、策略参数、因子参数、信号参数、风控参数。
