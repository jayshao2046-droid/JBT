# JBT 独立提交与回滚策略

【签名】Atlas
【时间】2026-04-03 13:48
【设备】MacBook

## 1. 目标

确保每个 agent 的任务都能独立提交、独立上传、独立回滚，不发生“全局回滚把别人工作一起回掉”的情况。

## 2. 强制规则

1. 每个任务必须独立提交。
2. 每个任务的提交范围只能覆盖自己的文件白名单。
3. 每个任务审核通过并锁回后，必须上传。
4. 回滚只能按任务提交或任务提交集合定向 revert。
5. 禁止以全局 reset、全局 checkout 覆盖方式回滚整个项目。

## 3. 推荐命名

### 分支命名

- `agent/模拟交易/TASK-0001`
- `agent/决策/TASK-0002`

### 提交信息

- `[模拟交易][TASK-0001] 建立订单 API 骨架`
- `[决策][TASK-0002] 补信号指令契约`

## 4. 上传时机

1. 任务通过项目架构师终审。
2. 文件已锁回。
3. 已生成回滚记录。
4. 已向 Jay.S 汇报并获得下一步确认。

满足以上四条后，才进入提交与上传。

## 5. 独立回滚方式

1. 先定位对应任务的提交 ID。
2. 只对该任务提交做定向 revert。
3. 更新 `docs/rollback/` 中的回滚记录。
4. 回滚完成后重新提交审核。

## 6. 禁止行为

1. 禁止回滚其他 agent 已通过审核的提交。
2. 禁止为了修一个任务把整个分支回退。
3. 禁止用“顺手合并修改”破坏任务边界。

## 7. 记录要求

每个任务必须至少有：

1. task 记录
2. review 记录
3. lock 记录
4. handoff 记录
5. rollback 记录
