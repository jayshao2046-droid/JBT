## 1. 问题来源

Mini 数据端健康报告（21:15）显示：
- 🔴 持仓周报（目录不存在）
- 🔴 宏观数据（无数据文件）
- 🔴 海运运费（无数据文件）
- 🔴 CFTC持仓（无数据文件）

## 2. 根因诊断

### 2.1 持仓路径不匹配（唯一真实路径错误）

| 项目 | 期望 | 实际 |
|------|------|------|
| health_check.py DATA_RULES dir | `position` | 应为 `position_daily` |
| pipeline.run_position_pipeline | symbol=`position_daily` | 写入 `/data/data/position_daily/position/records.parquet` |

`position_daily` 和 `position_weekly` 两条规则均指向 `DATA_ROOT / "position"`，而 pipeline 实际写入 `position_daily/`，导致目录始终"不存在"。

### 2.2 其余 3 项为时序问题（非路径错误）

| 项目 | 原因 |
|------|------|
| 宏观数据 | cron 为周一至周五 09:00，今天是周六 |
| 海运运费 | cron 为周一至周五 09:10，今天是周六 |
| CFTC持仓 | cron 为周六 10:00，但容器于 14:15 才启动，已错过 |

## 3. 修复内容

**修改文件**：`services/data/src/health/health_check.py`

```python
# 修复前
"position_daily": DataRule(label="持仓日报", dir=DATA_ROOT / "position", ...),
"position_weekly": DataRule(label="持仓周报", dir=DATA_ROOT / "position", ...),

# 修复后
"position_daily": DataRule(label="持仓日报", dir=DATA_ROOT / "position_daily", ...),
"position_weekly": DataRule(label="持仓周报", dir=DATA_ROOT / "position_daily", ...),
```

**仅改动 2 行，无逻辑变更，无跨服务影响。**
