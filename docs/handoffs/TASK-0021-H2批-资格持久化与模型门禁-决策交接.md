## 改动说明

1. `state_store.py` 继续复用 H1 已落地的本地 JSON 文件状态底座，只在同一状态文件中新增 `backtest_certs` 与 `research_snapshots` 两个 bucket；未新增数据库、缓存或任何新持久化栈。
2. `BacktestGate` 改为通过 `state_store` 读写回测证书，新增 `to_dict/from_dict` 序列化与 `get_backtest_gate()` 服务级单例工厂；默认直接构造仍走内存模式，避免破坏现有调用与测试。
3. `ResearchGate` 改为通过 `state_store` 读写研究快照，新增 `valid_until` 最小有效期字段、重启恢复能力与 `get_research_gate()` 服务级单例工厂；默认直接构造同样保留内存模式。
4. `model/router.py` 当前不再把“请求里有 `backtest_certificate_id` / `research_snapshot_id`”直接视为通过，而是读取策略仓库与两类 gate 的真实记录，拒绝以下情况：
   - 缺少有效回测证书；
   - 缺少完成态研究快照；
   - 请求携带的资格 ID 与当前活动资格不匹配；
   - 策略仓库与资格记录的因子版本哈希不匹配；
   - 回测资格与研究资格之间的因子版本哈希不一致。
5. `tests/test_gating.py` 新增 5 条 H2 定向用例，覆盖：
   - 回测资格持久化落盘；
   - 研究资格持久化落盘；
   - 重启后重新实例化 gate / repository 仍能恢复资格；
   - router 对缺失资格拒绝；
   - router 对不匹配资格拒绝；
   - router 对有效资格放行。
