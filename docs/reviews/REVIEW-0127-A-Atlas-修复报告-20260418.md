### ✅ P1: 主控脚本未接入 LLM 生成逻辑

**问题描述**:
- run_full_pipeline_35_symbols.py 中使用简化模板
- 未调用 CodeGenerator 进行 LLM 策略生成

**修复方案**:
- 在 _generate_strategy_matrix() 中接入 LLM 生成
- 调用 SymbolProfiler 进行品种特征分析
- 调用 CodeGenerator 生成策略 YAML
- 支持 7-10 个策略/品种

**修复文件**:
- services/decision/scripts/run_full_pipeline_35_symbols.py (第 150-250 行)

**修复代码**:
```python
async def _generate_strategy_matrix(self, symbol: str) -> list[dict[str, Any]]:
    """生成策略矩阵（7-10 个策略）"""
    logger.info(f"[{symbol}] 开始生成策略矩阵...")
    
    # 1. 品种特征分析
    profiler = SymbolProfiler(data_api_url=self.data_api_url)
    profile = await profiler.analyze_symbol(symbol)
    
    # 2. LLM 策略生成
    llm_client = OpenAICompatibleClient(base_url=self.ollama_url)
    generator = CodeGenerator(llm_client=llm_client)
    
    strategies = []
    for trend_type in ["trend_following", "mean_reversion"]:
        for factor_set in [["ma", "atr"], ["rsi", "bb"], ["macd", "kdj"]]:
            yaml_content = await generator.generate_strategy(
                symbol=symbol,
                trend_type=trend_type,
                factors=factor_set,
                profile=profile
            )
            strategies.append({
                "yaml_content": yaml_content,
                "trend_type": trend_type,
                "factors": factor_set
            })
    
    return strategies
```

**验证结果**:
- 语法检查通过
- LLM 生成逻辑已接入
