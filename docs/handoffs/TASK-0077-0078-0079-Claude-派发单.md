## TASK-0077 — CS1-S 容灾交接API（sim-trading）

**Token ID:** `tok-629df929`  
**白名单:** 4 文件  

### 背景

decision 端 `LocalSimEngine`（`services/decision/src/research/local_sim_engine.py`）在 sim-trading 不可用时本地接管模拟交易。当 sim-trading 恢复后，需要交接 API 接收积累的临时订单、持仓和账本差异。

LocalSimEngine 的数据结构：
- `orders: dict[str, LocalSimOrder]` — 每个 order 有 order_id, symbol, direction, quantity, price, status, created_at, filled_at
- `positions: dict[str, float]` — symbol → quantity
- `capital: float` — 当前资金

### 需要创建/修改的文件

**1. `services/sim-trading/src/failover/__init__.py`**（新建）
- 空 init 文件

**2. `services/sim-trading/src/failover/handler.py`**（新建）
- `FailoverHandler` 类，实现：
  - `receive_handover(data: HandoverPayload) -> HandoverResult` — 接收交接数据，校验完整性，合并到本地账本
  - `get_status() -> FailoverStatus` — 返回当前交接状态（idle/receiving/completed/error）
  - `confirm_handover(handover_id: str) -> bool` — 确认交接完成，清除临时状态
- Pydantic 模型：`HandoverPayload`（orders list, positions dict, capital float, source_engine str, handover_id str）
- Pydantic 模型：`HandoverResult`（success bool, handover_id str, orders_received int, positions_merged int, capital_delta float, errors list）
- Pydantic 模型：`FailoverStatus`（status str, last_handover_id Optional[str], last_handover_time Optional[str], orders_pending int）

**3. `services/sim-trading/src/api/router.py`**（修改）
- 在现有路由末尾添加 3 个端点：
  - `POST /api/v1/failover/handover` → 调用 handler.receive_handover
  - `GET /api/v1/failover/status` → 调用 handler.get_status
  - `POST /api/v1/failover/confirm` → 调用 handler.confirm_handover
- 注意：API Key 认证已在 `main.py` 全局中间件处理，router.py 不需要认证逻辑

**4. `services/sim-trading/tests/test_failover.py`**（新建）
- 至少 8 个测试用例：
  - 正常交接（有订单+持仓）
  - 空数据交接
  - 重复 handover_id 交接
  - 数据校验失败（负数量）
  - get_status 初始状态
  - get_status 交接后状态
  - confirm 正常确认
  - confirm 不存在的 handover_id

### commit 格式
```
feat(sim-trading): TASK-0077 CS1-S 容灾交接API
```
