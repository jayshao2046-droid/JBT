## 一、整体定位

### 新增 3 个核心页面

1. **登录页**
2. **首页 / 总控台**
3. **系统设置页**

这三个页面不是替代四端现有页面，而是作为**统一入口层**。

### 关系建议
- 登录后先进入：首页
- 首页作为全系统导航中枢
- 系统设置页负责全局配置
- 四端页面继续承担各自专业功能：
  - **sim-trading**：交易执行 / 风控 / 行情
  - **backtest**：回测 / 审核 / 参数优化
  - **data**：采集 / 质量 / 存储 / 数据源
  - **decision**：研究 / 信号 / 因子 / LLM / 发布

---

## 二、信息架构建议

建议做成 **五层结构**：

### 1. 访客层
- 登录页
- 注册/会员开通入口
- 忘记密码
- 产品介绍简版

### 2. 用户层
- 首页
- 个人中心
- 基础设置
- 通知偏好
- 查看自己有权限的服务页面

### 3. 高级会员层
- 可进入更多高级模块
- 可启用高级策略研究/回测分析/通知模板
- 可配置更复杂的提醒与观察项

### 4. 管理员层
- 用户管理
- 角色管理
- 会员等级管理
- 系统设置
- 通知模板管理
- 服务开关与交易时段控制

### 5. 超级管理员层
- 所有管理员能力
- 安全设置
- API 密钥管理
- 审计日志
- 全局只读/维护模式
- 服务降级控制

## 三、登录页设计方案

### 页面目标
让用户快速、安全进入系统，并明确当前是一个"多服务量化工作台"。

### 页面布局建议

#### 左侧
品牌与系统说明：
- JBT Logo
- 系统标题：JBT 量化控制台
- 副标题：交易 / 决策 / 数据 / 回测 一体化工作台
- 3~4 条亮点说明：
  - 实时交易监控
  - 策略研究与信号生成
  - 数据质量与采集治理
  - 回测审核与参数优化

#### 右侧
登录卡片区

---

### 登录方式建议

#### 方案 A：账号密码登录
- 手机号 / 邮箱 / 用户名
- 密码
- 记住我
- 登录按钮

#### 方案 B：会员码 / 邀请码登录
适合你的会员体系：
- 会员账号
- 密码
- 邀请码 / 激活码（可选）

#### 方案 C：管理员快捷入口
- 管理员登录入口单独按钮
- 避免普通用户默认看到后台管理能力

---

### 登录页功能清单

#### 基础功能
- 账号登录
- 忘记密码
- 记住登录状态
- 登录失败提示
- 图形验证码 / 滑块验证码
- 多端登录提醒

#### 安全功能
- 2FA 二次验证（短信 / 邮箱验证码）
- 异地登录提醒
- 登录设备识别
- 连续失败锁定
- 登录审计日志

#### 会员相关
- 会员等级显示说明
- 开通会员入口
- 试用会员入口
- 到期提醒

---

### 登录后跳转逻辑建议

- 普通用户：跳转首页
- 管理员：跳转首页，但首页出现"管理中心"区块
- 超级管理员：跳转首页，并可见全局系统状态卡片
- 若上次停留在某个服务页，可支持"恢复上次工作区"

## 四、首页设计方案

首页不要做成纯欢迎页，而要做成**总控台 / 运营首页**。

---

### 首页核心定位

首页 = **全系统总览 + 快速入口 + 今日重点 + 告警中枢**

用户一登录就应该知道：

1. 系统现在是否正常
2. 哪个服务有问题
3. 哪些告警最重要
4. 当前交易是否开启
5. 今天是不是交易日
6. 自己最常用入口在哪里
7. 自己的会员等级和权限范围是什么

---

### 首页建议分成 8 个模块

---

### 模块 1：顶部全局状态条

显示：
- 当前用户头像 / 会员等级 / 角色
- 当前时间
- 是否交易日
- 当前交易时段状态
- 系统总状态：正常 / 告警 / 维护中
- 通知状态：飞书 / 邮件 / 浏览器推送是否正常

#### 示例字段
- 用户名：Jay / 普通会员 / 管理员
- 当前模式：实盘关闭 / 模拟开启
- 交易时段：已开启 / 已关闭
- 今日是否交易日：是 / 否
- 当前告警数：P0 / P1 / P2

---

### 模块 2：四端总览卡片

四张大卡片：
1. 模拟交易
2. 回测
3. 数据
4. 决策

每张卡片显示：
- 服务在线状态
- 最近更新时间
- 核心 KPI
- 当前异常数量
- 快捷进入按钮

#### 示例

##### 模拟交易卡片
- 在线状态
- 今日盈亏
- 当前持仓数
- 风控状态
- CTP 连接状态

##### 回测卡片
- 今日回测次数
- 运行中任务
- 待审核策略数
- 最优策略收益率

##### 数据卡片
- 数据完整率
- 采集器运行状态
- 最新数据延迟
- 缺失告警数

##### 决策卡片
- 今日信号数
- 研究任务数
- 发布待审批数
- LLM Pipeline 状态

---

### 模块 3：今日重点 / 待处理事项

这是首页最重要的运营区。

显示：
- 待审核策略
- 数据缺失待处理
- 风控告警待确认
- 信号发布待审批
- 回测失败任务
- 通知发送失败任务

#### 支持功能
- 一键跳转到对应页面
- 按优先级排序
- 按服务筛选
- 显示负责人 / 创建时间 / 状态

---

### 模块 4：全局告警中心

统一聚合四端告警：
- sim-trading 风控告警
- backtest 回测失败告警
- data 数据完整性告警
- decision 发布/信号/模型告警

#### 建议分级
- P0：立即处理
- P1：高优先级
- P2：一般
- P3：提示

#### 建议能力
- 告警确认
- 告警静音
- 告警转发
- 查看详情
- 跳转源页面

---

### 模块 5：快捷操作区

不同角色看到不同快捷入口。

#### 普通用户
- 进入模拟交易
- 进入回测中心
- 查看数据质量
- 进入研究中心
- 查看通知中心

#### 会员用户
- 新建回测
- 导入策略
- 创建研究任务
- 配置告警
- 导出报告

#### 管理员
- 用户管理
- 角色管理
- 系统设置
- 通知模板
- 审计日志

---

### 模块 6：交易日历与交易时段状态

显示：
- 今日是否交易日
- 当前属于日盘/夜盘/休市
- 下一个交易时段开始时间
- 是否允许下单
- 是否允许策略执行
- 是否允许自动通知

#### 可视化建议
做成一张时段卡片：
- 日盘
- 夜盘
- 休市
- 节假日状态

---

### 模块 7：通知与消息中心

统一聚合：
- 飞书通知发送状态
- 邮件发送状态
- 浏览器提醒状态
- 用户系统消息
- 审批消息
- 会员消息

#### 支持
- 未读/已读
- 按类型筛选
- 一键全部已读
- 消息跳转原始页面

---

### 模块 8：个人工作台

显示当前用户个人维度信息：
- 我的角色
- 我的会员等级
- 到期时间
- 我的常用页面
- 我的最近访问
- 我的收藏页面
- 我的提醒订阅

## 七、权限模型建议

建议采用 **RBAC + 会员等级双轨制**

### RBAC 决定"能做什么"
例如：
- 普通用户：只能看
- 交易员：可操作 sim-trading
- 研究员：可操作 decision/backtest
- 管理员：可管理设置
- 超管：全权限

### 会员等级决定"能看到哪些高级功能"
例如：
- 基础会员：只能看基础看板
- 高级会员：可看高级 KPI 和导出
- 专业会员：可用高级研究/回测分析
- 企业管理员：可配置团队与权限
