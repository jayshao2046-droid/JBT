## API 规格冻结

### 1. 拉取端点（已有消费代码）

**路由**：`GET /api/v1/researcher/report/latest`

**请求**：
```bash
curl http://192.168.31.156:8105/api/v1/researcher/report/latest \
  -H "Authorization: Bearer $DATA_API_KEY"
```

**响应**：
```json
{
  "id": "2026-04-21-14-00",
  "date": "2026-04-21",
  "timestamp": "2026-04-21T14:00:00Z",
  "news": [
    {
      "title": "期货市场开盘涨跌幅扩大",
      "content": "...",
      "source": "eastmoney_futures",
      "published_at": "2026-04-21T13:50:00Z"
    }
  ],
  "macro": {
    "key_indicators": [...],
    "sentiment": "neutral"
  },
  "confidence": 0.72,
  "reviewed": true
}
```

**响应码**：
- 200：成功（返回最新研报 JSON）
- 204：无研报（返回 `{}` 或 null）
- 401：认证失败
- 500：服务错误

**数据来源**：
- 从 Alienware 8199 研究员服务调用 API 或 SCP 读取文件
- 建议方案：Mini 定期从 Alienware 8199 拉取最新研报缓存（TTL 1h）

### 2. 推送端点（可选，便于 Alienware 主动推送）

**路由**：`POST /api/v1/researcher/reports`

**请求体**：
```json
{
  "report_id": "2026-04-21-14-00",
  "data": { /* 完整研报 JSON */ }
}
```

**响应**：
```json
{
  "status": "ok",
  "report_id": "2026-04-21-14-00",
  "cached_at": "2026-04-21T14:00:05Z"
}
```

## 实现方案选项

### 方案 A：Mini 被动拉取（推荐）
- Mini 接收 `/api/v1/researcher/report/latest` 请求后，从 Alienware 8199 拉取最新研报
- 本地缓存 1 小时（key=`researcher:report:latest`，TTL=3600s）
- 实现位置：`services/data/src/main.py` 新增路由 + 后台任务刷新缓存

### 方案 B：Alienware 主动推送
- Alienware 研究员在生成研报后，主动 POST 到 Mini `/api/v1/researcher/reports`
- Mini 接收后存入内存缓存
- 实现位置：同上，新增 POST 路由接收推送

### 方案 C：混合模式
- 优先接收 Alienware 主动推送（快速路径）
- 若 5 分钟无推送，Mini 主动从 Alienware 8199 拉取（容灾路径）
