# 双回测验证流程方案

**文档版本**: v1.0  
**创建时间**: 2026-04-18  
**目标**: 确保策略在 YamlSignalExecutor 和 TqSDK 两个回测引擎中都能正常运行，并通过双回测报告验证策略有效性

---

## 1. 流程概述

```
┌─────────────────────────────────────────────────────────────────┐
│                     策略生成与优化完整流程                        │
└─────────────────────────────────────────────────────────────────┘

Step 1: 品种特征分析
  ↓
  SymbolProfiler.calculate_features()
  → 输出: runtime/symbol_profiles/{symbol}.json

Step 2: 策略生成
  ↓
  ThreeTierOptimizer.generate_strategy_for_symbol()
  → 输出: runtime/three_symbols_strategies/{symbol}_strategy.yaml

Step 3: 本地回测（YamlSignalExecutor）
  ↓
  YamlSignalExecutor.backtest_strategy()
  → 输出: runtime/local_backtest_report.json
  → 验证: Sharpe ≥ 1.5, 交易次数 ≥ 30, 胜率 ≥ 45%

Step 4: TqSDK 标准回测
  ↓
  TqsdkBacktester.backtest_strategy()
  → 输出: runtime/tqsdk_backtest_report.json
  → 验证: 与本地回测结果对比，偏差 < 10%

Step 5: 双回测报告对比
  ↓
  BacktestComparator.compare_reports()
  → 输出: runtime/dual_backtest_comparison.json
  → 验证: 关键指标一致性检查

Step 6: 策略调优（如果需要）
  ↓
  MetaOptimizer.optimize_strategy()
  → 输出: 优化后的策略 YAML
  → 重复 Step 3-5

Step 7: 策略上线
  ↓
  策略通过双回测验证，可以部署到实盘
```

## 2. YAML 策略格式要求

### 2.1 核心原则

**重要**：我们的 YAML 格式是**自定义格式**，不是 TqSDK 原生格式。TqSDK 使用纯 Python 代码，我们通过桥接层将 YAML 转换为 TqSDK 代码。

### 2.2 YAML 格式定义

```yaml
# ========== 基本信息 ==========
name: "rb_trend_strategy_v1"
description: "螺纹钢趋势跟踪策略"
version: "1.0"

# ========== 交易品种 ==========
symbols:
  - "SHFE.rb2505"                    # 主力合约

# ========== K线周期 ==========
timeframe_minutes: 120               # 2小时K线

# ========== 技术因子 ==========
factors:
  - factor_name: "ATR"
    params:
      period: 14
  - factor_name: "ADX"
    params:
      period: 14
  - factor_name: "RSI"
    params:
      period: 14

# ========== 信号条件 ==========
signal:
  long_condition: "atr > 0.015 * close and adx > 25 and rsi > 50"
  short_condition: "atr < 0.008 * close or adx < 20 or rsi < 50"
  confirm_bars: 2

# ========== 市场过滤 ==========
market_filter:
  enabled: true
  conditions:
    - "atr > 0.005 * close"
    - "adx > 15"

# ========== 仓位管理 ==========
position_fraction: 0.1               # 10% 仓位

# ========== 风险控制 ==========
risk:
  stop_loss_yuan: 1000               # 止损1000元
  take_profit_yuan: 3000             # 止盈3000元
  daily_loss_limit_yuan: 2000        # 日最大亏损2000元
  max_drawdown_pct: 0.015            # 最大回撤1.5%
  force_close_time: "14:55"          # 强制平仓时间
  no_overnight: true                 # 禁止过夜

# ========== 交易成本 ==========
transaction_costs:
  slippage_per_unit: 1               # 滑点1跳
  commission_per_lot_round_turn: 5   # 手续费5元/手
```

### 2.3 字段说明

| 字段 | 类型 | 必填 | 说明 |
|-----|------|------|------|
| `name` | string | ✅ | 策略唯一标识 |
| `symbols` | list | ✅ | 交易品种列表 |
| `timeframe_minutes` | int | ✅ | K线周期（分钟） |
| `factors` | list | ✅ | 技术因子配置 |
| `signal.long_condition` | string | ✅ | 多头入场条件 |
| `signal.short_condition` | string | ✅ | 空头入场条件 |
| `position_fraction` | float | ✅ | 仓位比例（0-1） |
| `transaction_costs` | dict | ✅ | 交易成本配置 |
| `risk` | dict | ✅ | 风险控制参数 |

### 4.2 TqSDK 桥接层工作原理

```python
# 1. 读取 YAML 配置
with open(yaml_path, 'r') as f:
    strategy = yaml.safe_load(f)

# 2. 初始化 TqSDK API
from tqsdk import TqApi, TqAuth, TqSim, TqBacktest

api = TqApi(
    TqSim(500000),  # 初始资金50万
    backtest=TqBacktest(
        start_dt=datetime(2024, 1, 1),
        end_dt=datetime(2024, 12, 31)
    ),
    auth=TqAuth("17621181300", "Jay.486858")
)

# 3. 获取K线数据
symbol = strategy['symbols'][0]
timeframe = strategy['timeframe_minutes'] * 60
klines = api.get_kline_serial(symbol, duration_seconds=timeframe)

# 4. 计算技术因子
from tqsdk.ta import ATR, ADX, RSI

factors = {}
for factor_config in strategy['factors']:
    if factor_config['factor_name'] == 'ATR':
        factors['atr'] = ATR(klines, factor_config['params']['period'])
    elif factor_config['factor_name'] == 'ADX':
        factors['adx'] = ADX(klines, factor_config['params']['period'])
    # ... 其他因子

# 5. 执行回测逻辑
while True:
    api.wait_update()
    
    # 评估信号条件
    env = {
        'close': klines.close.iloc[-1],
        'atr': factors['atr'].iloc[-1],
        'adx': factors['adx'].iloc[-1],
        # ...
    }
    
    # 检查开仓条件
    if eval(strategy['signal']['long_condition'], env):
        # 下单逻辑
        pass
```

## 5. 双回测报告对比

### 5.1 对比工具

```python
class BacktestComparator:
    """双回测报告对比器"""
    
    def compare_reports(
        self,
        local_report_path: str,
        tqsdk_report_path: str
    ) -> dict:
        """对比两个回测报告"""
        
        with open(local_report_path) as f:
            local = json.load(f)
        with open(tqsdk_report_path) as f:
            tqsdk = json.load(f)
        
        # 对比关键指标
        comparison = {
            "sharpe_ratio": {
                "local": local['performance']['sharpe_ratio'],
                "tqsdk": tqsdk['performance']['sharpe_ratio'],
                "diff": abs(local['performance']['sharpe_ratio'] - tqsdk['performance']['sharpe_ratio']),
                "diff_pct": abs(local['performance']['sharpe_ratio'] - tqsdk['performance']['sharpe_ratio']) / local['performance']['sharpe_ratio']
            },
            "total_return": {
                "local": local['performance']['total_return'],
                "tqsdk": tqsdk['performance']['total_return'],
                "diff": abs(local['performance']['total_return'] - tqsdk['performance']['total_return']),
                "diff_pct": abs(local['performance']['total_return'] - tqsdk['performance']['total_return']) / local['performance']['total_return']
            },
            "trades_count": {
                "local": local['performance']['trades_count'],
                "tqsdk": tqsdk['performance']['trades_count'],
                "diff": abs(local['performance']['trades_count'] - tqsdk['performance']['trades_count'])
            },
            "win_rate": {
                "local": local['performance']['win_rate'],
                "tqsdk": tqsdk['performance']['win_rate'],
                "diff": abs(local['performance']['win_rate'] - tqsdk['performance']['win_rate'])
            }
        }
        
        # 判断是否通过验证
        passed = (
            comparison['sharpe_ratio']['diff_pct'] < 0.1 and  # Sharpe偏差<10%
            comparison['total_return']['diff_pct'] < 0.1 and  # 收益偏差<10%
            comparison['trades_count']['diff'] <= 5 and        # 交易次数偏差≤5
            comparison['win_rate']['diff'] < 0.05              # 胜率偏差<5%
        )
        
        return {
            "passed": passed,
            "comparison": comparison,
            "summary": self._generate_summary(comparison, passed)
        }
```

### 5.2 验证标准

| 指标 | 允许偏差 | 说明 |
|-----|---------|------|
| Sharpe 比率 | < 10% | 风险调整后收益一致性 |
| 总收益率 | < 10% | 绝对收益一致性 |
| 交易次数 | ≤ 5 次 | 信号触发一致性 |
| 胜率 | < 5% | 盈利交易比例一致性 |
| 最大回撤 | < 15% | 风险控制一致性 |

## 6. 完整流程脚本

### 6.1 自动化脚本

```bash
#!/bin/bash
# 文件: scripts/run_dual_backtest.sh

SYMBOL=$1
START_DATE=${2:-"2024-01-01"}
END_DATE=${3:-"2024-12-31"}

echo "=========================================="
echo "双回测验证流程"
echo "=========================================="
echo "品种: $SYMBOL"
echo "回测期间: $START_DATE ~ $END_DATE"
echo ""

# Step 1: 生成策略
echo "[1/5] 生成策略..."
python -c "
from services.decision.src.research.three_tier_optimizer import ThreeTierOptimizer
import asyncio

async def main():
    optimizer = ThreeTierOptimizer()
    result = await optimizer.generate_strategy_for_symbol('$SYMBOL')
    print(f'策略已生成: {result[\"strategy_yaml_path\"]}')

asyncio.run(main())
"

STRATEGY_PATH="runtime/three_symbols_strategies/${SYMBOL}_strategy.yaml"

# Step 2: 本地回测
echo "[2/5] 执行本地回测..."
python -c "
from services.decision.src.research.yaml_signal_executor import YamlSignalExecutor
import asyncio
import json

async def main():
    executor = YamlSignalExecutor()
    result = await executor.backtest_strategy(
        '$STRATEGY_PATH',
        '$START_DATE',
        '$END_DATE'
    )
    
    with open('runtime/local_backtest_report.json', 'w') as f:
        json.dump(result, f, indent=2, ensure_ascii=False)
    
    print(f'本地回测完成: Sharpe={result[\"sharpe_ratio\"]:.2f}, 收益={result[\"total_return\"]:.2%}')

asyncio.run(main())
"

# Step 3: TqSDK 回测
echo "[3/5] 执行 TqSDK 回测..."
python scripts/backtest_with_tqsdk_final.py \
  $STRATEGY_PATH \
  --start $START_DATE \
  --end $END_DATE \
  > runtime/tqsdk_backtest_report.json

# Step 4: 对比报告
echo "[4/5] 对比双回测报告..."
python -c "
from scripts.compare_backtest_reports import BacktestComparator
import json

comparator = BacktestComparator()
result = comparator.compare_reports(
    'runtime/local_backtest_report.json',
    'runtime/tqsdk_backtest_report.json'
)

with open('runtime/dual_backtest_comparison.json', 'w') as f:
    json.dump(result, f, indent=2, ensure_ascii=False)

print(result['summary'])

if not result['passed']:
    print('❌ 双回测验证失败！需要检查策略逻辑或回测引擎差异。')
    exit(1)
"

# Step 5: 生成最终报告
echo "[5/5] 生成最终报告..."
python scripts/generate_final_report.py \
  --local runtime/local_backtest_report.json \
  --tqsdk runtime/tqsdk_backtest_report.json \
  --comparison runtime/dual_backtest_comparison.json \
  --output runtime/${SYMBOL}_final_report.md

echo ""
echo "=========================================="
echo "✅ 双回测验证完成！"
echo "=========================================="
echo "最终报告: runtime/${SYMBOL}_final_report.md"
```

## 7. 策略调优流程

### 7.1 调优触发条件

如果双回测验证**未通过**，或者策略表现**不达标**，需要进入调优流程：

**不达标标准**：
- Sharpe < 1.5
- 交易次数 < 30
- 胜率 < 45%
- 最大回撤 > 8%
- 双回测偏差 > 10%

### 7.2 调优方法

```python
from services.decision.src.research.meta_optimizer import MetaOptimizer

optimizer = MetaOptimizer()

# 执行参数优化
result = await optimizer.optimize_strategy(
    symbol='rb',
    strategy_path='runtime/three_symbols_strategies/rb_strategy.yaml',
    optimization_rounds=5,
    metric='sharpe_ratio'
)

# 保存优化后的策略
optimized_strategy_path = result['optimized_strategy_path']

# 重新执行双回测
bash scripts/run_dual_backtest.sh rb 2024-01-01 2024-12-31
```

### 7.3 调优迭代

```
初始策略 → 双回测 → 不达标？
                ↓ 是
            参数优化 → 双回测 → 不达标？
                            ↓ 是
                        策略重构 → 双回测 → 达标！
                                        ↓
                                    策略上线
```

## 8. 总结

### 8.1 关键要点

1. **YAML 格式是自定义的**：不是 TqSDK 原生格式，通过桥接层转换
2. **双回测是必需的**：确保策略在两个引擎中都能正常运行

4. **调优是迭代的**：不达标就优化，直到通过验证
