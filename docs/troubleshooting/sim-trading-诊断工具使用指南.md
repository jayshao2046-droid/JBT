# Sim-Trading 诊断工具使用指南

## 问题 1：追踪飞书推送来源

### 目标
反向追踪每一条飞书通知的真实来源，找出是否有定时任务在推送 TRADE_EXECUTED。

### 使用步骤

#### 1. 在 Alienware 上启动追踪模式

```bash
# SSH 到 Alienware
ssh jayshao@192.168.31.223

# 进入 JBT 目录
cd ~/JBT

# 停止当前运行的 sim-trading
pkill -f "sim-trading"
# 或者
ps aux | grep main.py | grep -v grep | awk '{print $2}' | xargs kill

# 启动追踪模式（会 hook FeishuNotifier.send 方法）
python3 scripts/trace_feishu_sender.py
```

#### 2. 等待收集数据

让脚本运行至少 **1 个交易日**，覆盖：
- 上午盘：09:00 - 11:30
- 下午盘：13:00 - 15:00
- 夜盘：21:00 - 23:00

追踪日志会实时写入：`/tmp/feishu_trace.jsonl`

#### 3. 分析追踪日志

```bash
# 在 MacBook 或 Alienware 上运行
python3 scripts/analyze_feishu_trace.py
```

- **如果时间分布随机**：是真实交易触发

## 问题 2：诊断 CTP 断联原因

### 目标
深度分析 CTP 断联的根本原因：网络问题、服务器问题、还是配置问题。

### 使用步骤

#### 1. 在 Alienware 上启动诊断模式

```bash
# SSH 到 Alienware
ssh jayshao@192.168.31.223

# 进入 JBT 目录
cd ~/JBT

# 启动诊断脚本（不影响现有 sim-trading 进程）
python3 scripts/diagnose_ctp_disconnect.py
```

**注意**：这个脚本会读取现有 gateway 状态，不需要停止 sim-trading。

#### 2. 等待收集数据

让脚本运行至少 **24 小时**，覆盖：
- 交易时段
- 非交易时段
- 周末

诊断日志会实时写入：`/tmp/ctp_disconnect_diagnosis.jsonl`

#### 3. 分析诊断日志

```bash
# 在 MacBook 或 Alienware 上运行
python3 scripts/analyze_ctp_disconnect.py
```

## 常见问题

### Q1: 追踪脚本会影响性能吗？
**A**: 会有轻微影响（每次飞书推送多写一次文件），但不影响交易。建议在非高频交易时段运行。

### Q2: 诊断脚本需要 root 权限吗？
**A**: 不需要。但需要能访问 sim-trading 进程的 gateway 对象。

### Q3: 日志文件会占用多少空间？
**A**: 
- `feishu_trace.jsonl`：约 1KB/条，运行 1 天约 500KB
- `ctp_disconnect_diagnosis.jsonl`：约 2KB/条，运行 1 天约 100KB

### Q4: 如何清理日志？
```bash
rm /tmp/feishu_trace.jsonl
rm /tmp/ctp_disconnect_diagnosis.jsonl
```

### Q5: 可以远程分析吗？
**A**: 可以。把日志文件从 Alienware 拷贝到 MacBook：
```bash
scp jayshao@192.168.31.223:/tmp/feishu_trace.jsonl /tmp/
scp jayshao@192.168.31.223:/tmp/ctp_disconnect_diagnosis.jsonl /tmp/
python3 scripts/analyze_feishu_trace.py
python3 scripts/analyze_ctp_disconnect.py
```

## 下一步

### 如果确认是定时任务推送
1. 检查 `src/main.py` 的 `_report_scheduler` 和 `_ctp_connection_guardian`
2. 检查是否有其他定时任务（cron、systemd timer）
3. 修改代码，移除不必要的推送

### 如果确认是 CTP 心跳超时
1. **接受现状**：非交易时段断联是正常的，SimNow 可能有空闲超时限制
2. **降低告警级别**：把非交易时段的 `P1` 改成 `P2` 或完全静默
3. **增加心跳频率**（需要修改 CTP 代码，不推荐）

### 如果确认是网络问题
1. 检查 Alienware 网卡状态：`ifconfig` / `ip addr`
2. 检查路由：`traceroute 180.168.146.187`
3. 检查防火墙：`sudo iptables -L` (Linux) / `pfctl -s rules` (macOS)
4. 检查 DNS：`nslookup 180.168.146.187`
