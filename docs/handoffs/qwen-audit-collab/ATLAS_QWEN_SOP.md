## 总体节奏

```
[Atlas] 出 Batch-N 任务 → 弹窗通知 Jay.S
   ↓ Jay.S 确认 + 通知 Qwen
[Qwen] 执行 Batch-N（多个子任务）
   ↓ 同时
[Atlas] 向架构师提交白名单预审 / 向 Jay.S 申请 Token / 预建 Batch-N+1 档案
   ↓ Qwen 完成，Jay.S 把结果发给 Atlas
[Atlas] 评分 + 审核 + 补漏（不低于 100 分再推进）
   ↓
[Atlas] 弹窗通知 Jay.S + 交出下一批任务
[Jay.S] 把 Batch-N+1 发给 Qwen，同时 Atlas 完成补漏/锁回
```

## 快速复核通道（加速规则）

当 Qwen 自评分 ≥ 95 且自查清单全绿时，Atlas 启用**快速复核**：
1. 只抽查 §2 锚点声明（文件是否真实存在）
2. 只核对 §7 依赖链与 Step3 §4 图一致性
3. 复核时间目标：≤ 5 分钟
4. 通过后直接输出架构师预审材料，不做全文逐行审查

当 Qwen 自评分 < 95 或自查清单有未确认项时，走完整复核（全 5 维度）。

## 评分标准（满分 100，低于 80 暂停上报）

| 维度 | 分值 | 说明 |
|------|------|------|
| 完整性 | 20 | 所有必填字段无缺项，所有子任务均有产出 |
| 锚点真实性 | 20 | 不得将 planned-placeholder 写成已存在锚点 |
| 接口清晰度 | 20 | 方法签名、字段名、schema 清晰无歧义 |
| 依赖链准确性 | 20 | 依赖关系与 Atlas 审定的 Step3 编排保持一致 |
| 测试设计 | 20 | 覆盖 happy path + 边界 + 异常路径 |

**评分规则**：
- 每发现一处 planned-placeholder 误标 → 扣 5 分
- 每发现一处依赖链错误 → 扣 5 分
- 每发现一处必填字段缺失 → 扣 3 分
- 低于 80 分：必须暂停，向 Jay.S 汇报详细扣分原因，等待指示

## Qwen 交付质量要求（强制）

1. **零瑕疵原则**：每次提交前 Qwen 必须自查，确认无 planned-placeholder 误标、无依赖链错误、无必填字段缺项。
2. **锚点声明义务**：每个实施规格书必须在 "锚点声明" 章节明确列出哪些文件已存在、哪些是新建、哪些是 planned-placeholder。
3. **依赖闭环义务**：每个任务发出前 Qwen 必须读取 `QWEN_PHASE_C_STEP3_REPORT.md` § 4 并行关系图，确保依赖声明与图一致。
4. **Atlas 无需重写**：Qwen 的输出文档应达到 Atlas 可直接用于架构师预审的质量，Atlas 只做补充或格式调整，不做实质性内容重写。
