## 五、问题分析

### 问题根因

CTP 穿透式监控要求在登录（`ReqUserLogin`）之前，先调用 `ReqAuthenticate` 进行客户端认证。该接口需要填入以下字段：

```
CThostFtdcReqAuthenticateField:
    BrokerID  = "6000"       ← 已有
    UserID    = "00001920"   ← 已有
    AppID     = ???          ← 【缺失】
    AuthCode  = ???          ← 【缺失】
```

我们**没有正确的 AppID 和 AuthCode**，因此交易前置返回 ErrorID=63（客户端认证失败），后续登录因未通过认证而返回 ErrorID=64（客户端未认证）。

### 当前缺口

| 缺失信息 | 说明 |
|----------|------|
| **AppID**（终端软件代码 / 应用标识） | 用于标识接入软件，需在中期协穿透式监管系统注册 |
| **AuthCode**（客户端认证码 / 授权码） | 期货公司为该 AppID 签发的认证密钥 |
