### 场景 1.1: 股票因子加载功能验证

- **操作步骤**:
  1. 调用 FactorLoader.load_stock_bars 方法，传入有效的股票代码和时间范围
  2. 验证返回的 DataFrame 包含正确的字段和数据
  3. 检查数据格式是否符合预期
- **预期结果**: 成功返回包含 datetime, open, high, low, close, volume, open_interest 字段的 DataFrame
- **验收标准**: 数据完整性 100%，字段格式正确，无异常抛出

- **前提条件**: data 服务短暂不可用或响应缓慢
- **操作步骤**:
  1. 模拟 data 服务超时或返回错误
  2. 触发 FactorLoader 的股票数据加载
  3. 观察重试行为和最终结果
- **预期结果**: 客户端执行最多 3 次重试，使用指数退避策略
- **验收标准**: 重试机制正常工作，不会无限重试，超时设置生效

### 场景 1.3: 错误处理机制验证
- **前提条件**: data 服务返回非预期格式或无效数据
- **操作步骤**:
  1. 模拟 data 服务返回格式错误的响应
  2. 调用 FactorLoader.load_stock_bars 方法
  3. 检查异常处理逻辑
- **预期结果**: 抛出相应的 DataParseError 或 FactorLoadError 异常
- **验收标准**: 异常类型正确，错误信息清晰，不会导致程序崩溃

### 场景 1.4: 期货数据兼容性验证
- **前提条件**: FactorLoader 已扩展支持股票数据
- **操作步骤**:
  1. 调用原有的 load_futures_bars 方法
  2. 验证期货数据加载功能未受影响
  3. 比较新旧功能的表现
- **预期结果**: 期货数据加载功能正常工作，与股票功能互不影响
- **验收标准**: 期货数据加载无回归，股票和期货功能各自正常

### 场景 1.5: 性能基准测试
- **前提条件**: 大量股票数据可供加载
- **操作步骤**:

  3. 验证缓存机制的有效性
- **预期结果**: 在合理时间内完成数据加载，资源消耗在可接受范围内
- **验收标准**: 单次加载时间 < 5秒，内存使用稳定，缓存命中率 > 80%

- **操作步骤**:

- **预期结果**: 返回包含绩效指标的详细结果摘要
- **验收标准**: 结果数据完整，绩效指标计算准确，格式符合 API 规范

- **前提条件**: 回测已完成并处于 completed 状态
- **操作步骤**:
  1. 通过 POST /api/v1/backtest/approval/{run_id}/approve 提交审核
  2. 验证状态从 completed 转换为 approved
  3. 检查审核记录的完整性
- **预期结果**: 状态成功更新为 approved，审核信息被记录
- **验收标准**: 状态转换正确，审核信息完整，无数据丢失

### 场景 3.4: 人工审核拒绝验证
- **前提条件**: 回测已完成并处于 completed 状态
- **操作步骤**:
  1. 通过 POST /api/v1/backtest/approval/{run_id}/reject 提交拒绝
  2. 提供拒绝理由

- **预期结果**: 状态成功更新为 rejected，拒绝理由被记录
- **验收标准**: 状态转换正确，拒绝理由保存完整，审核流程闭环

### 场景 3.5: 异常边界情况验证
- **前提条件**: 各种异常状态组合
- **操作步骤**:
  1. 尝试对非 completed 状态的项目进行审核
  2. 尝试重复审核已审核的项目
