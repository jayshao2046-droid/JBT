## 1. 概述

JBT 研究中心是一个全自动化的量化研究系统，提供从数据采集、特征计算、策略生成到回测验证的完整闭环。本文档介绍如何使用研究中心的各项自动化功能。

### 1.1 核心能力

- **24/7 数据采集**：自动采集期货、股票、宏观、新闻等多源数据
- **智能特征计算**：自动计算品种统计特征（波动率、趋势强度等）
- **策略自动生成**：基于品种特征自动生成交易策略
- **回测自动验证**：自动回测策略并生成评估报告
- **研究报告生成**：Alienware 研究员 24/7 生成市场研究报告

### 1.2 四设备架构

| 设备 | 角色 | 核心功能 |
|------|------|---------|
| **Mini** | 数据源 + 快速投喂节点 | 数据采集、存储、API 服务 |
| **Studio** | 决策/开发主控节点 | 策略生成、回测、决策引擎 |
| **Alienware** | Windows 交易端 + 研究员节点 | 研究报告生成、交易软件 |
| **Air** | 回测生产节点 | 大规模回测计算 |

## 3. 品种特征自动化

### 3.1 自动更新机制

**调度时间**：每日 17:30（交易日）  
**更新品种**：35 个期货主力连续合约  
**更新方式**：增量更新（只获取新增的日K线数据）  

**品种列表**：
```
上期所(17): cu0, al0, zn0, pb0, ni0, sn0, au0, ag0, rb0, wr0, hc0, ss0, fu0, bu0, ru0, sp0, sc0
大商所(16): c0, cs0, a0, b0, m0, y0, p0, l0, v0, pp0, j0, jm0, i0, eg0, eb0, pg0
郑商所(2):  SR0, CF0
```

### 3.2 特征维度

每个品种自动计算以下特征：

| 特征类型 | 时间窗口 | 说明 |
|---------|---------|------|
| **波动率** | 3个月/1年/5年 | 年化标准差，分级为 Low/Medium/High |
| **趋势强度** | 3个月/1年 | ADX 指标，分级为 Weak/Medium/Strong |
| **自相关性** | 3个月/1年 | 收益率序列自相关系数 |
| **偏度** | 1年 | 收益率分布偏度 |
| **峰度** | 1年 | 收益率分布峰度 |
| **流动性** | 3个月 | 成交量分级 |

### 3.3 特征缓存机制

**缓存目录**：`runtime/symbol_profiles/`

**缓存结构**：
```json
{
  "symbol": "SHFE.rb0",
  "last_update": "2026-04-18T17:30:00",
  "features": {
    "volatility_3m": 0.0134,
    "volatility_1y": 0.0161,
    "volatility_weighted": "Medium",
    "trend_strength_3m": 0.0000,
    "trend_strength_1y": 0.0000,
    "trend_strength_weighted": "Weak",
    ...
  },
  "confidence": {
    "overall": 0.98,
    "volatility": 0.95,
    "trend": 0.90,
    ...
  }
}
```

**性能优势**：
- 首次计算：~5秒（获取5年数据）
- 增量更新：~0.5秒（只获取1天数据）
- 性能提升：**6-10倍**

## 4. 策略生成自动化

### 4.1 策略生成流程

```
品种特征 → 参数映射 → 策略生成 → 回测验证 → 评分排序
```

### 4.2 参数自适应映射

**波动率映射**：
```python
# Low 波动率品种（如国债）
ATR_MULTIPLIER: 1.5-2.0
STOP_LOSS_ATR: 2.0-2.5

# Medium 波动率品种（如螺纹钢）
ATR_MULTIPLIER: 2.0-3.0
STOP_LOSS_ATR: 2.5-3.0

# High 波动率品种（如铁矿石）
ATR_MULTIPLIER: 3.0-4.0
STOP_LOSS_ATR: 3.0-4.0
```

**趋势强度映射**：
```python
# Weak 趋势品种
CONFIRM_BARS: 2-3
MA_PERIOD: 20-30

# Medium 趋势品种
CONFIRM_BARS: 1-2
MA_PERIOD: 15-25

# Strong 趋势品种
CONFIRM_BARS: 1
MA_PERIOD: 10-20
```

### 4.3 策略模板

**YAML 标准模板位置**：
```
services/decision/src/research/templates/strategy_template.yaml
```

**模板用途**：
- CodeGenerator 自动生成策略时使用此模板
- 确保所有生成的策略包含完整的回测参数
- 保证策略可以直接进入回测系统和 TqSDK

**模板包含的关键参数**：
```yaml
# 资金管理
initial_capital: 500000           # 初始资金（元）
position_fraction: 0.1            # 单次开仓比例

# 风险控制
risk:
  stop_loss_yuan: 1000            # 单笔止损（元）
  take_profit_yuan: 3000          # 单笔止盈（元）
  daily_loss_limit_yuan: 2000     # 日最大亏损（元）
  max_drawdown_pct: 0.015         # 最大回撤（1.5%）
  consecutive_loss_pause_days: 1  # 连续亏损暂停天数
  max_position_per_day: 1         # 每日最大开仓次数
  time_to_exit_bars: 12           # 强制平仓时间（K线数）
  margin_usage_percent: 15        # 最大保证金使用率

# 交易成本
transaction_costs:
  slippage_per_unit: 1            # 滑点（元/单位）
  commission_per_lot_round_turn: 5 # 手续费（元/手往返）
```

**支持的策略类型**：
1. **波动率突破策略**：适用于 Medium/High 波动率品种
2. **均线交叉策略**：适用于 Strong 趋势品种
3. **通道突破策略**：适用于 Weak 趋势品种
4. **动量策略**：适用于高自相关性品种

**重要说明**：
- ✅ 所有生成的策略都会包含完整的风控参数
- ✅ 可以直接用于 YamlSignalExecutor 回测
- ✅ 可以转换为 TqSDK 格式进行回测
- ⚠️ `initial_capital` 当前版本硬编码为50万，未来版本将支持YAML配置

## 5. 回测自动化

### 5.1 回测引擎

**位置**：Studio 设备（192.168.31.142:8103）

**支持的回测类型**：
1. **单策略回测**：验证单个策略的历史表现
2. **参数优化回测**：寻找最优参数组合
3. **批量回测**：同时回测多个策略

### 5.2 回测指标

| 指标类型 | 指标名称 | 说明 |
|---------|---------|------|
| **收益指标** | 总收益率 | 期末资金 / 初始资金 - 1 |
| | 年化收益率 | 总收益率 / 年数 |
| **风险指标** | 最大回撤 | 最大资金回撤百分比 |
| | Sharpe 比率 | 年化收益 / 年化波动率 |
| **交易指标** | 交易次数 | 总交易笔数 |
| | 胜率 | 盈利交易 / 总交易 |
| | 盈亏比 | 平均盈利 / 平均亏损 |

### 5.3 手动触发回测

```bash
# 单策略回测
curl -X POST "http://192.168.31.142:8103/api/v1/backtest/run" \
  -H "Content-Type: application/json" \
  -d '{
    "strategy_file": "strategies/rb_volatility_breakout_001.yaml",
    "start_date": "2024-01-01",
    "end_date": "2026-04-18"
  }'

# 批量回测
python scripts/backtest_all_strategies.py
```

## 9. 故障排查

### 9.1 数据采集失败

**症状**：飞书健康报告显示红灯

**排查步骤**：
1. 查看容器日志：`docker logs JBT-DATA-8105 --tail 100`
2. 检查网络连接：`ping api.tushare.pro`
3. 检查 API Token：`echo $TUSHARE_TOKEN`
4. 手动触发采集：`python -m src.scheduler.data_scheduler --once`

**常见原因**：
- API Token 过期
- 网络连接问题
- API 限流
- 数据源 API 变更

### 9.2 品种特征更新失败

**症状**：缓存目录无新文件

**排查步骤**：
1. 查看调度日志：`docker logs JBT-DATA-8105 | grep "品种特征"`
2. 手动触发更新：`python scripts/update_symbol_features.py`
3. 检查 Mini API：`curl http://192.168.31.76:8105/health`
4. 检查缓存目录权限：`ls -la runtime/symbol_profiles/`

**常见原因**：
- Mini API 服务未启动
- 日K线数据未采集
- 缓存目录权限问题
- Python 依赖缺失

### 9.3 研究报告生成失败

**症状**：Alienware 报告目录无新文件

**排查步骤**：
1. 检查服务状态：`curl http://192.168.31.223:8199/health`
2. 查看队列状态：`curl http://192.168.31.223:8199/queue/status`
3. 手动触发：`curl -X POST http://192.168.31.223:8199/run`
4. 查看日志：`type D:\researcher_reports\logs\researcher.log`

**常见原因**：
- Ollama 服务未启动
- qwen3:14b 模型未加载
- Mini API 连接失败
- 磁盘空间不足

### 9.4 回测失败

**症状**：回测 API 返回错误

**排查步骤**：
1. 检查服务状态：`curl http://192.168.31.142:8103/health`
2. 检查策略文件：`cat strategies/xxx.yaml`
3. 检查数据可用性：`curl "http://192.168.31.76:8105/api/v1/bars?symbol=SHFE.rb0&start=2024-01-01&end=2026-04-18&interval=1440"`
4. 查看回测日志：`tail -f ~/jbt/services/backtest/logs/backtest.log`

**常见原因**：
- 策略 YAML 格式错误
- 历史数据不足
- 回测引擎内存不足
- 参数超出合理范围

## 10. 最佳实践

### 10.1 数据使用

1. **优先使用 Mini API**：避免直接读取 Parquet 文件
2. **缓存热数据**：频繁访问的数据应缓存到本地
3. **批量请求**：避免频繁的小请求，使用批量接口
4. **错误重试**：网络请求应有重试机制

### 10.2 策略开发

1. **基于特征生成**：使用品种特征自动映射参数

3. **参数优化**：使用回测引擎寻找最优参数
