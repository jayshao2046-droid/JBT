## 二、实施内容

### 2.1 队列管理机制

**新增文件**：`services/data/src/researcher/queue_manager.py`

**功能**：
- 维护三种状态：pending（待读）、processing（处理中）、completed（已完成）
- 线程安全的状态转换
- JSONL 格式持久化（`.queue/pending.jsonl`、`processing.jsonl`、`completed.jsonl`）
- 支持查询、标记、清理操作

**关键方法**：
```python
add_to_queue(report_id, file_path, metadata)  # 添加到待读队列
mark_processing(report_id)                     # 标记为处理中
mark_completed(report_id, result)              # 标记为已完成
get_pending(limit)                             # 获取待读列表
get_status(report_id)                          # 查询状态
```

### 2.2 报告生成集成

**修改文件**：`services/data/src/researcher/reporter.py`

**改动**：
1. 初始化队列管理器
2. `_save_classified_report()` 方法增强：
   - 报告落档后自动调用 `queue_manager.add_to_queue()`
   - 增强日志：记录 report_id、类型、路径、品种数
   - 异常处理：保存失败时记录详细错误

**日志示例**：
```
[REPORT_SAVE] 报告已保存: RPT-20260417-14-futures -> D:\researcher_reports\2026-04-17\futures_14.json (类型=futures, 品种数=35)
```

### 2.3 研究员服务 API

**修改文件**：`services/data/run_researcher_server.py`

**新增端点**：

1. **GET /reports/latest**
   - 功能：获取最新待读报告
   - 返回：报告 JSON 数据
   - 供 Studio decision 调用

2. **POST /reports/{report_id}/mark_read**
   - 功能：标记报告为已读
   - 参数：score、reasoning、model
   - 供 Studio 评级完成后调用

3. **GET /queue/status**
   - 功能：查询队列状态
   - 返回：pending_count、processing_count、待读报告列表

**日志示例**：
```
[API] 读取最新报告: RPT-20260417-14-futures
[API] 标记已读: RPT-20260417-14-futures, score=85.0, model=qwen3:14b
```

### 2.4 评级器增强

**修改文件**：`services/decision/src/llm/researcher_phi4_scorer.py`

**改动**：
1. 新增 `researcher_api_url` 参数（默认 `http://192.168.31.187:8199`）
2. 新增 `_mark_report_read()` 方法：
   - 评级完成后自动调用 Alienware API 标记已读
   - 传递 score、reasoning、model
3. 增强 `score_report()` 日志：
   - 开始评级：记录 report_id
   - 完成评级：记录 score、model

**日志示例**：
```
[SCORER] 开始评级报告: RPT-20260417-14-futures
[SCORER] 评级完成: RPT-20260417-14-futures, score=85.00, model=qwen3:14b
[SCORER] 标记已读成功: RPT-20260417-14-futures
```
