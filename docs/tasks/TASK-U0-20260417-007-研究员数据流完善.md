## 1. 问题诊断

### 当前问题
1. **路径混乱**：
   - `REPORTS_DIR` 默认 `D:\researcher_reports`（Windows）
   - 部分代码硬编码 `/data/researcher/`（Linux）
   - Mini data 路径未统一指向宿主机

2. **缺少状态管理**：
   - 无"待读队列"机制
   - 无"已读标记"功能
   - qwen3 读取后无状态更新

3. **日志不完整**：
   - 采集 → 落档：缺少日志
   - 落档 → 通知：缺少日志
   - 通知 → 读取：缺少日志
   - 读取 → 标记：缺少日志

## 2. 目标架构

### 完整数据流
```
[1] Mini 采集数据
    ↓ 落档到宿主机 /Users/jaybot/JBT/data/{symbol}/1min/records.parquet
    ↓ 日志：data_scheduler.py
    
[2] Mini 回调 Alienware
    ↓ HTTP POST http://192.168.31.187:8199/callback
    ↓ 日志：data_scheduler.py + run_researcher_server.py
    
[3] Alienware 读取 Mini 数据
    ↓ 从 Mini API 拉取：http://192.168.31.74:8105/api/v1/bars
    ↓ 日志：staging.py
    
[4] Alienware 生成研报
    ↓ qwen3:14b 分析
    ↓ 落档到：D:\researcher_reports\{date}\{report_type}_{hour}.json
    ↓ 写入待读队列：D:\researcher_reports\.queue\pending.jsonl
    ↓ 日志：reporter.py + scheduler.py
    
[5] Alienware 推送到 Studio
    ↓ HTTP POST http://192.168.31.142:8104/api/v1/evaluate
    ↓ 日志：scheduler.py
    
[6] Studio qwen3 读取研报
    ↓ 从 Alienware 读取：http://192.168.31.187:8199/reports/latest
    ↓ 或直接读取文件：\\192.168.31.187\researcher_reports\{date}\{file}
    ↓ 日志：researcher_phi4_scorer.py
    
[7] Studio qwen3 评级
    ↓ qwen3:14b 评分
    ↓ 标记已读：HTTP POST http://192.168.31.187:8199/reports/{id}/mark_read
    ↓ 日志：researcher_phi4_scorer.py
    
[8] Studio 飞书通知
    ↓ 发送评级结果
    ↓ 日志：feishu.py
```
