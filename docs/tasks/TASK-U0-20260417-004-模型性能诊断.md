## 问题描述

Studio 上的策略调优流水线使用本地 Ollama 模型（deepcoder:14b、qwen3:14b）生成策略代码，但遇到严重性能问题：

1. **deepcoder:14b**: 请求超时（300秒），无法完成代码生成
2. **qwen3:14b**: 已运行4分钟仍无响应，性能同样不佳
3. **phi4-reasoning:14b**: 用于代码审核，性能未测试

## 根本原因

本地 14B 参数模型在 Studio (Mac Mini M2) 上推理速度过慢，无法满足策略调优流水线的性能要求（每个策略需要多次 LLM 调用）。

### ✅ Researcher ↔ Critic 迭代优化流程设计（2026-04-17 19:30）

**流程架构**：
```
1. Researcher (qwen-coder-plus) 生成初始代码
   ↓
2. 回测引擎执行 → 返回指标
   ↓
3. 检查是否满足最低标准？
   - 是 → 完成
   - 否 → Critic (qwen3:14b) 诊断
   ↓
4. Critic 输出诊断 + 调整建议（JSON）
   ↓
5. Researcher 根据反馈重新生成代码
   ↓
6. 重复 2-5（最多3轮）
```

**最低标准**（防止浪费 Token）：
- OOS 交易次数 ≥ 10（放宽到10）
- OOS Sharpe ≥ 0.5（放宽到0.5）
- OOS 胜率 ≥ 40%（放宽到40%）
- 最大回撤 ≤ 5%

**底线约束**（绝对不能违反）：
- 必须保持日内交易（no_overnight: true）
- 单笔止损 ≤ 1000 元
- 日内止损 ≤ 2000 元
- 不能移除核心因子（MACD/RSI/VolumeRatio）

### ✅ Critic 提示词设计（2026-04-17 19:35）
