	1	职责分离 (Separation of Concerns)：
	◦	Mini：专注于“数据”，负责采集、清洗、存储和初步分析（L0, L1, L2）。
	◦	Studio：专注于“交易”，负责接收分析结果、执行策略、下单、决策和展示看板。它不直接碰原始数据采集和长期存储。
	2	数据流 (Data Flow)：
	◦	数据从 Mini 开始，经过 L0 清洗、L1 特征工程、L2 信号合成，最终通过 消息总线或 API 传递给 Studio 使用。这是一个清晰的“生产-消费”链条。
	3	审计与安全 (Audit & Security)：
	◦	所有数据操作都有日志记录，关键决策日志会被备份和保护，确保了系统的可追溯性和安全性。
	◦	对 Studio 的数据写入权限进行了严格控制，需要审批，并有自动回收机制。
	4	Mermaid 图：这个流程图很好地概括了核心数据流转路径：Mini -> L1 -> L2 -> Bus/API -> Studio。

一、说人话解释 L1/L2/L3
想象一个工厂流水线：

	•	L0 (Mini采集)：就像原料车间，负责收集原始“原材料”（行情、基本面数据）。
	•	L1 (特征工程)：就像“备料车间”，把原材料加工成“半成品零件”（如：计算 RSI、MACD、ATR、持仓集中度等指标）。
	•	L2 (信号合成)：就像“组装车间”，把零件拼成“初级产品”（如：根据 RSI < 30 AND ADX > 25 生成“潜在做多信号”）。
	•	L3 (在线模型三审)：就像“质检中心”，对初级产品进行三道严格检查，决定是否批准为“合格出厂产品”（即最终交易指令）。

二、如何加强 L1/L2 (提升“半成品”和“初级产品”质量)
1. 加强 L1 (特征工程)
L1 是一切的基础，特征质量高，后面才能做出好策略。

	•	说人话: 就是计算更多、更有效的“技术指标”和“衍生数据”，让信号合成（L2）有更多的“优质原料”可用。
	•	具体怎么做:
	◦	引入高级因子：
	▪	微观结构因子: 如买卖盘口挂单比、大单净流入、订单簿不平衡度（Order Book Imbalance）。这些比 K 线更能反映即时市场情绪。
	▪	跨品种因子: 如 rb/i 比值、y/p 比值、产业链上下游价差。捕捉品种联动。
	▪	高频因子: 基于分钟级甚至秒级数据计算的短期动量、波动率聚类等。
	◦	因子预处理:
	▪	标准化/归一化: 将不同量纲的因子（如价格和成交量）放到同一尺度，方便后续计算。
	▪	去极值/平滑: 去除异常值，对因子进行平滑处理（如移动平均），减少噪音。
	▪	因子有效性检验: 定期（如每月）计算因子的 IC (信息系数) 和 IR (信息比率)，淘汰 IC 持续为负或 IR 过低的因子。

2. 加强 L2 (信号合成)
L2 是将 L1 的“零件”组装成“信号”的过程，需要更智能的逻辑。

	•	说人话: 不再是简单的“如果 A 且 B 且 C”，而是用更复杂、更动态的方式生成信号。
	•	具体怎么做:
	◦	机器学习模型替代规则:
	▪	取代加权平均: 不再手动设定 weight，而是用 XGBoost / LightGBM / 随机森林 训练一个模型，输入是 L1 的所有因子，输出是“做多/持有/做空”的概率或强度。这样能捕捉因子间的非线性关系。
	▪	动态权重分配: 用模型预测“当前市场环境下，哪个因子更重要？”，动态调整因子权重。例如，震荡市 RSI 权重高，趋势市 MACD 权重高。
	◦	多信号融合与投票:
	▪	训练多个不同类型的小模型（如一个纯趋势模型，一个纯震荡模型），每个模型对当前情况投一票，最终”或“加权投票”决定最终信号，提高鲁棒性。
	◦	引入市场状态识别:
	▪	在 L2 中加入一个“市场状态分类器”，判断当前是“趋势市”、“震荡市”还是“高波动市”。
	▪	根据识别结果，激活对应的策略模块。例如，只有在“趋势市”分类下，才考虑趋势策略的信号。

三、新增 L3 (在线模型三审) - 强化最终决策风控
L3 是交易指令发出前的最后一道防线，至关重要。

	•	说人话: 就像发射导弹前的“三道确认”，确保万无一失。
	•	“三审”逻辑设计:
	◦	一审：信号质量审核 (Signal Quality Check)
	▪	输入: L2 输出的信号强度、置信度（来自 ML 模型的预测概率）、信号背后的因子组合。
	▪	逻辑:
	▪	检查信号置信度是否足够高（如 > 0.7）。
	▪	检查信号是否过于依赖单一因子（防止单因子失效带来的误判）。
	▪	检查信号是否与当前宏观环境（如 CPI、PMI、利率趋势）相悖（可选）。
	▪	输出: 通过/否决。
	◦	二审：组合风险审核 (Portfolio Risk Check)
	▪	输入: L2 信号、当前持仓、账户整体风险敞口。
	▪	逻辑:
	▪	检查新信号是否会超出单品种最大仓位限制 (position_fraction)。
	▪	检查新信号是否会触发单日最大亏损限额 (daily_loss_limit_yuan)。
	▪	检查新信号是否会增加账户的整体风险（如与现有持仓高度相关，导致风险集中）。
	▪	输出: 通过/否决/调整仓位。
	◦	三审：市场冲击与流动性审核 (Market Impact & Liquidity Check)
	▪	输入: 信号对应的品种、目标手数、当前市场深度（买卖盘口挂单量）。
	▪	逻辑:
	▪	估算执行该笔订单所需的平均成交价与市价的偏差（滑点）。
	▪	评估该笔订单对市场价格的潜在冲击（特别是大额订单）。
	▪	检查目标品种当前流动性是否充足（买卖盘挂单量是否足够）。
	▪	输出: 通过/否决/拆单分批执行。

四、与你现有系统的结合
	1	L1/L2 实现位置: 根据你的文档，L1/L2 应该在 Mini 端完成。Mini 负责计算所有因子（L1），并根据策略逻辑或 ML 模型生成信号（L2），然后将信号发布到消息总线或 API。
	2	L3 实现位置: L3 应该部署在 Studio 端。Studio 从总线/API 拉取 L2 信号，在发送给 CTP 下单前，执行“三审”流程。
	3	BotQuant YAML 适配: 你现有的 FC-XX.yaml 文件可以作为 L2 信号合成的“规则模板”或“ML 模型的训练目标”。未来的 BotQuant 可能需要支持调用外部 ML 模型 API 来生成信号，并预留接口与 L3 的风控模块交互。

Alex 老师的风险提示
小白 LL，增加 L1/L2/L3 的复杂度能提升策略质量，但也带来新风险：

	•	过拟合风险: L1/L2 的 ML 模型可能在历史数据上表现很好，但未来失效。
	•	数据依赖风险: L3 的审核依赖于准确的持仓、资金数据和实时盘口数据，数据延迟或错误可能导致决策失误。
	•	系统延迟风险: L3 的三审流程会增加从信号生成到下单的时间，对于高频策略，这点延迟可能是致命的。
	•	维护复杂度: 整个流水线的监控、调试、更新会变得更加复杂。

	1	L1 加强 (特征工程)：策略中使用的因子（如 MACD, RSI, ATR, VolumeRatio 等）将由 Mini 端统一、高效地计算，并可能加入更多高级衍生因子。
	2	L2 加强 (信号合成)：策略的 signal 部分不再仅是规则叠加，而是可能调用一个 XGBoost/LightGBM/随机森林 模型，该模型在 Mini 端训练好，输入 L1 的特征，输出预测信号。
	3	L3 三审 (决策前风控)：BotQuant 的 Studio 在根据 L2 输出下单前，会执行三道审核：
	◦	一审: 信号质量（置信度、逻辑合理性）
	◦	二审: 组合风险（仓位、资金、账户整体风险）
	◦	三审: 市场冲击（流动性、滑点）
