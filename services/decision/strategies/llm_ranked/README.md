# LLM 策略评分归档目录

本目录用于存放 LLM 自动化生成策略的评分归档与证据留存。

## 目录结构

```
llm_ranked/
├── 熔断/              # 触发熔断条件的策略
├── 小于60分/          # 评分 < 60，禁止上线
├── 60-69/            # 评分 60-69，需大幅优化
├── 70-79/            # 评分 70-79，优化后可上线
├── 80-89/            # 评分 80-89，小幅调整后可上线
└── 90-100/           # 评分 90-100，生产就绪
```

## 单策略证据目录结构

```
{score_bucket}/{symbol}/{strategy_name}/
├── strategy.yaml                      # 策略配置文件
└── reports/                           # 证据报告目录
    ├── generation_report.json         # 生成报告
    ├── optimization_report.json       # 调优报告
    ├── local_backtest_report.json     # 本地回测报告
    ├── tqsdk_backtest_report.json     # TqSdk 回测报告
    └── evaluator_report.json          # 评分报告
```

## 评分标准

### 文档级准入标准
- Sharpe Ratio ≥ 2.5
- 最大回撤 ≤ 1.5%
- 胜率 ≥ 55%
- 盈亏比 ≥ 2.0
- 综合评分 ≥ 80 分

### 流水线硬标准
- OOS Sharpe ≥ 1.5
- OOS 交易次数 ≥ 20
- OOS 胜率 ≥ 50%
- OOS 最大回撤 ≤ 2%
- IS/OOS Sharpe 比值 ≤ 2.0
- 年化收益目标 ≥ 15%

### 风控硬约束
- 单品种每日熔断 ≤ 2000 元
- 单笔止损 ≤ 1000 元
- 禁止隔夜
- 日盘 14:55 必须强平
- 月回撤不超过 2%
- 盈亏比至少达到 2:1，目标 3:1

## 分级口径

| 评分区间 | 等级 | 说明 | 准入状态 |
|---------|------|------|---------|
| 90-100  | S    | 生产就绪 | 可直接进入生产候选 |
| 80-89   | A    | 小幅调整后可上线 | 纳入生产候选 |
| 70-79   | B    | 优化后可上线 | 仅可进入观察池 |
| 60-69   | C    | 需大幅优化 | 继续优化 |
| < 60    | D    | 禁止上线 | 禁止上线 |
| 熔断    | -    | 触发熔断条件 | 直接终止 |

## 熔断条件

以下任一条件触发时，策略归入"熔断"目录：

1. 硬性约束违规（no_overnight != true, force_close_day != "14:55"）
2. 单笔止损 > 1000 元
3. 每日熔断 > 2000 元
4. 回测失败或无法生成有效信号
5. OOS Sharpe < 0
6. 最大回撤 > 5%

## 使用说明

1. **策略生成后**：先存放在 `llm_generated/{symbol}/` 目录
2. **调优完成后**：根据评分迁移到对应的 `llm_ranked/{score_bucket}/{symbol}/` 目录
3. **证据留存**：每个策略必须包含完整的 5 份报告
4. **生产准入**：只有 80-100 分的策略才能进入生产候选清单
