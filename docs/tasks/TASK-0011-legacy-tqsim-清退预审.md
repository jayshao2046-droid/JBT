- 文档类型：独立运维任务预审
- 签名：项目架构师
- 建档时间：2026-04-06

## 一、任务目标

把 legacy Studio 上现有 tqsim 运行态的清退动作从 JBT 服务实现中彻底分离，作为独立运维任务管理，并先冻结清退前置条件、回滚点要求与授权门槛。

本任务当前只做预审与建档，不执行任何清理、停机、删除、容器调整或 Studio 运维动作。

---

## 二、任务编号与归属结论

### 编号结论

- **本事项独立建档为 `TASK-0011`。**

### 判定理由

1. 本事项不是 JBT 代码实现，而是 legacy 运行态处置。
2. 其执行窗口、回滚点、授权链条与 `services/sim-trading/**` 完全不同。
3. 若把该事项混入 `TASK-0010`，会把 Git 白名单与运行态动作错误绑定在一起，违反“一件事一审核一上锁”。

### 执行归属

1. 预审与验收：项目架构师
2. 实际运维执行：Atlas（且必须在 Jay.S 明确运维授权后）

---

## 三、当前轮次边界

### 当前 Git 白名单

- **无。**

### 当前轮次明确禁止

1. 不得把 legacy 清退动作并入任何 `services/sim-trading/**` Token。

---

## 四、清退前置条件冻结

`TASK-0011` 只能在以下条件全部满足后进入执行态：

1. `TASK-0010` 已完成最小服务骨架并终审通过。
2. JBT SimNow 健康检查可返回 `200`。
3. JBT SimNow 已具备并验证最小风险门闸。
4. `TASK-0014` 已完成最小告警通道验证。
5. JBT SimNow 已具备并验证只减仓模式。
6. `TASK-0017` 已完成 Mini 部署与切换前验证。
7. `TASK-0012` 或 `TASK-0016` 至少存在一条已验证的上游信号接入路径。
8. Atlas 已先向 Jay.S 汇报清退窗口、回滚点与影响范围，并获得明确运维授权。

说明：

1. 上述任一条件未满足，legacy tqsim 只能维持现状，不得提前切流或停机。
2. 即使 JBT SimNow 已可启动，只要只减仓、最小告警、Mini 部署验证或上游信号接入路径未完成，也不得清退 legacy tqsim。

---

## 五、回滚点与执行前强制动作

1. 实际清退前，必须先对当前 legacy tqsim 容器执行 `docker commit`，形成明确可回滚镜像。
2. 回滚镜像必须记录：源容器名、镜像 tag、创建时间、执行人、对应任务号。
3. 在未完成 `docker commit` 前，不得停止、删除或重建 legacy tqsim 容器。
4. 实际 stop / rm / disable / 清理动作必须作为单独运维步骤逐条确认，不得批量模糊执行。

---

## 六、验收与失败收口原则

1. 清退任务验收不以“容器已删掉”为唯一标准，而以“JBT SimNow 已稳定接管 + legacy 回滚点已保留 + 运行窗口已受控”为标准。
2. 若清退后发现 JBT SimNow 不满足最小健康、只减仓或告警要求，必须优先回滚 legacy，而不是继续在线修补。
3. 清退失败时只允许按本任务保留的回滚点恢复，不得借机改动 JBT 服务代码。

---
