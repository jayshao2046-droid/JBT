### 2.3 phi4 评级器实现

**文件**: `services/decision/src/llm/researcher_phi4_scorer.py` (新建)

**核心功能**:
- 使用 phi4-reasoning:14b 模型对研报进行真实评级
- 评分标准：100 分制
  - 数据完整性 (50%) - 是否包含期货、股票、新闻数据
  - 信息质量 (30%) - 数据是否真实、分析是否有深度
  - 时效性 (20%) - 报告是否及时、信息是否新鲜

**评分等级**:
- 80-100分: 优秀，数据完整且有价值
- 60-80分: 良好，数据基本完整
- 40-60分: 一般，数据不完整
- 20-40分: 较差，缺少关键数据
- 0-20分: 很差，几乎无数据

**输出格式** (JSON):
```json
{
  "score": 85,
  "reasoning": "简短评价（1-2句话）",
  "improvements": ["改进建议1", "改进建议2"]
}
```

**关键方法**:
- `score_report(report, context)`: 评估报告并返回评分
- `_calculate_confidence(score, report)`: 计算置信度 (high/medium/low)
- `_extract_score_from_text(text)`: 从文本中提取评分（降级处理）

**技术细节**:
- 自动清理 phi4 输出中的 `<think>` 标签
- 支持中文和英文字段名（"评分"/"score"）
- JSON 解析失败时降级为文本提取

## 三、技术要点

### 3.1 依赖变更

**问题**: 本地 `queue` 模块与 Python 标准库冲突

**解决**: 将 `services/decision/src/queue` 目录重命名为 `task_queue`

**影响文件**: 所有导入 `queue` 的模块需更新为 `task_queue`

### 3.2 JSON 序列化

**问题**: phi4 输出包含 `<think>` 标签，导致 JSON 解析失败

**解决**: 使用正则表达式清理：
```python
import re
content = re.sub(r'<think>.*?</think>', '', content, flags=re.DOTALL)
```

### 3.3 评分系统

**从 0-1 改为 0-100**:
- 系统提示词：`"score": 85` 而非 `"score": 0.85`
- 置信度计算：`score >= 70` 而非 `score >= 0.7`
- 飞书卡片：`{score:.0f}/100` 而非 `{score:.2f}/1.0`

## 五、后续工作建议

### 5.1 短期优化

1. **定时评级任务**
   - 在 Studio 决策端添加定时任务，每小时自动评估最新研报
   - 使用 cron 或 APScheduler 实现

2. **评级历史记录**
   - 将评级结果存储到数据库
   - 支持评级趋势分析和报表生成

3. **评分阈值告警**
   - 当评分低于阈值（如 40 分）时发送告警通知
   - 提醒研究员系统可能存在数据采集问题

### 5.2 中期改进

1. **多维度评级**
   - 扩展评级维度：数据覆盖率、信息新鲜度、分析深度
   - 为每个维度单独打分

2. **评级对比**
   - 对比同一时段不同日期的研报质量
   - 识别研报质量趋势（改善/下降）

3. **自动反馈循环**
   - 将 phi4 的改进建议自动反馈给研究员系统
   - 优化数据采集和分析策略

### 5.3 长期规划

1. **研报质量预测**
   - 基于历史评级数据训练预测模型
   - 提前识别可能的数据采集问题

2. **多模型评级**
   - 使用多个 LLM 模型进行评级
   - 综合多个模型的评分提高准确性

3. **用户反馈机制**
   - 允许用户对 phi4 评级进行反馈
   - 使用反馈数据微调评级模型

## 六、注意事项

### 6.1 环境变量

确保 Studio 决策端配置以下环境变量：
```bash
FEISHU_WEBHOOK_URL=https://open.feishu.cn/open-apis/bot/v2/hook/xxx
```

### 6.2 网络连通性

- Studio 必须能访问 Alienware (192.168.31.224:8199)
- 确保防火墙规则允许该连接

### 6.3 模型依赖

- phi4-reasoning:14b 模型必须在 Ollama 中可用
- 如果模型不可用，系统会降级返回默认评分 50 分

### 6.4 性能考虑

- phi4 评级耗时约 50-60 秒
- 建议异步调用，避免阻塞主流程
- 可考虑添加缓存机制，避免重复评级
