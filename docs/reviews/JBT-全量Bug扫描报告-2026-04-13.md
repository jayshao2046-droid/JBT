## 1. SIM-TRADING 服务

### 1.1 全局状态竞态条件 ⚠️ CRITICAL

**文件**: `services/sim-trading/src/api/router.py`  
**行号**: 18, 477, 510, 556, 562  
**问题描述**: 
- `_gateway` 全局变量在多线程环境下存在竞态条件
- `ctp_connect()` 使用 `_connect_lock` 保护，但 `ctp_disconnect()` 直接修改 `_gateway` 而不加锁
- 其他路由（如 `get_positions()`, `get_orders()`, `get_ticks()`）读取 `_gateway` 时也未加锁

**影响范围**:
- 并发请求可能导致 `_gateway` 状态不一致
- 可能触发 `AttributeError: 'NoneType' object has no attribute 'xxx'`
- CTP 连接状态管理混乱

**修复建议**:
```python
# 在 ctp_disconnect() 中添加锁保护
@router.post("/ctp/disconnect")
def ctp_disconnect():
    global _gateway
    _connect_lock.acquire()
    try:
        if _gateway is not None:
            _gateway.disconnect()
        _gateway = None
        return {"status": "disconnected"}
    finally:
        _connect_lock.release()

# 在所有读取 _gateway 的地方添加锁保护
@router.get("/positions")
def get_positions():
    _connect_lock.acquire()
    try:
        if _gateway is None:
            raise HTTPException(status_code=503, detail="Gateway not connected")
        return _gateway.get_positions()
    finally:
        _connect_lock.release()
```

**优先级**: P0 - 立即修复

def _mark_signal_seen(signal_id: str) -> None:

            del _received_signal_ids[oldest]
        _received_signal_ids[signal_id] = True

## 2. DECISION 服务

### 2.1 信号分发器内存泄漏 🔴 HIGH

**文件**: `services/decision/src/core/signal_dispatcher.py`  
**行号**: 33, 43-51, 54-59, 75-78, 94-97  
**问题描述**:
- `self._dispatched` 字典无限增长，没有过期清理机制
- 每个信号都被永久保存在内存中
- 长期运行会导致内存溢出

**影响范围**:
- 服务运行 7x24 小时后内存占用可能达到 GB 级别
- 最终导致 OOM 崩溃

**修复建议**:
```python
from datetime import datetime, timedelta, timezone

class SignalDispatcher:
    def __init__(self, sim_trading_url: str = "http://localhost:8101", max_history: int = 10000) -> None:
        self.sim_trading_url = sim_trading_url.rstrip("/")
        self._dispatched: dict[str, dict] = {}
        self.max_history = max_history
        self._cleanup_threshold = timedelta(hours=24)
    
    def _cleanup_old_records(self):
        """清理超过 24 小时的记录"""
        now = datetime.now(timezone.utc)
        to_delete = []
        for signal_id, rec in self._dispatched.items():
            dispatched_at = rec.get("dispatched_at")
            if dispatched_at:
                try:
                    ts = datetime.fromisoformat(dispatched_at)
                    if now - ts > self._cleanup_threshold:
                        to_delete.append(signal_id)
                except:
                    pass
        
        for signal_id in to_delete:
            del self._dispatched[signal_id]
        
        # 如果仍然超过限制，按时间排序删除最旧的
        if len(self._dispatched) > self.max_history:
            sorted_items = sorted(
                self._dispatched.items(),
                key=lambda x: x[1].get("dispatched_at", "")
            )
            to_delete = [item[0] for item in sorted_items[:len(self._dispatched) - self.max_history]]
            for signal_id in to_delete:
                del self._dispatched[signal_id]
    
    async def dispatch(self, request: SignalDispatchRequest) -> SignalDispatchResponse:
        # 在每次分发前清理旧记录
        self._cleanup_old_records()
        # ... 原有逻辑
```

**优先级**: P0 - 立即修复

### 2.2 信号分发重试逻辑缺陷 🟡 MEDIUM

**文件**: `services/decision/src/core/signal_dispatcher.py`  
**行号**: 66-91  
**问题描述**:
- 重试时创建新的 `AsyncClient`，但没有连接池复用
- 每次重试都会创建新连接，浪费资源
- 异常捕获过于宽泛 `(httpx.HTTPStatusError, httpx.RequestError)`，可能隐藏其他错误

**影响范围**:
- 重试时创建大量 TCP 连接
- 可能触发操作系统文件描述符限制

**修复建议**:
```python
class SignalDispatcher:
    def __init__(self, sim_trading_url: str = "http://localhost:8101"):
        self.sim_trading_url = sim_trading_url.rstrip("/")
        self._dispatched: dict[str, dict] = {}
        self._client = httpx.AsyncClient(timeout=10)  # 类级别维护单一客户端
    
    async def dispatch(self, request: SignalDispatchRequest) -> SignalDispatchResponse:
        url = f"{self.sim_trading_url}/signal"
        payload = {
            "signal_id": request.signal_id,
            "strategy_id": request.strategy_id,
            "action": request.action,
            "symbol": request.symbol,
            "quantity": request.quantity,
            "price": request.price,
            "timestamp": request.timestamp,
        }
        
        for attempt in range(1, _MAX_RETRIES + 1):
            try:
                resp = await self._client.post(url, json=payload)
                resp.raise_for_status()
                data = resp.json()
                
                self._dispatched[request.signal_id] = {
                    "signal_id": request.signal_id,
                    "status": "dispatched",
                    "dispatched_at": datetime.now(timezone.utc).isoformat(),
                    "response": data,
                }
                
                return SignalDispatchResponse(
                    signal_id=request.signal_id,
                    status="dispatched",
                    message="Signal dispatched successfully",
                )
            
            except httpx.HTTPStatusError as exc:
                logger.warning(f"HTTP error on attempt {attempt}: {exc}")
                if attempt == _MAX_RETRIES:
                    raise
                await asyncio.sleep(_RETRY_DELAY * attempt)
            
            except httpx.RequestError as exc:
                logger.warning(f"Network error on attempt {attempt}: {exc}")
                if attempt == _MAX_RETRIES:
                    raise
                await asyncio.sleep(_RETRY_DELAY * attempt)
            
            except Exception as exc:
                logger.error(f"Unexpected error: {exc}", exc_info=True)
                raise
    
    async def close(self):
        """清理资源"""
        await self._client.aclose()
```

**文件**: `services/data/src/collectors/news_api_collector.py`  
**行号**: 36-50  
**问题描述**:
- `collect()` 方法尝试获取当前事件循环，如果失败则创建新循环
- 新循环创建后没有正确清理，可能导致资源泄漏
- 在已有事件循环的环境中调用会抛出 `RuntimeError`

**影响范围**:
- 在 FastAPI 异步路由中调用会失败
- 事件循环资源泄漏

**修复建议**:
```python
def collect(self, *, sources: list[str] | None = None, keywords: list[str] | None = None, as_of: str | None = None) -> list[dict[str, Any]]:
    source_list = sources or self.DEFAULT_SOURCES
    if self.use_mock:
        return self._mock_records(source_list=source_list, as_of=as_of)
    
    try:
        loop = asyncio.get_running_loop()
        # 已在事件循环中，不能使用 asyncio.run
        # 返回 coroutine 让调用者 await
        raise RuntimeError("Cannot use asyncio.run in running event loop. Use await collect_async() instead.")
    except RuntimeError as e:
        if "running event loop" in str(e):
            raise
        # 不在事件循环中，可以使用 asyncio.run
        try:
            return asyncio.run(self._async_fetch_all(source_list=source_list, as_of=as_of))
        except Exception as exc:
            self.logger.warning("news_api async fetch failed: %s", exc)
            return self._mock_records(source_list=source_list, as_of=as_of)

async def collect_async(self, *, sources: list[str] | None = None, keywords: list[str] | None = None, as_of: str | None = None) -> list[dict[str, Any]]:
    """异步版本，供 FastAPI 路由使用"""
    source_list = sources or self.DEFAULT_SOURCES
    if self.use_mock:
        return self._mock_records(source_list=source_list, as_of=as_of)
    
    try:
        return await self._async_fetch_all(source_list=source_list, as_of=as_of)
    except Exception as exc:
        self.logger.warning("news_api async fetch failed: %s", exc)
        return self._mock_records(source_list=source_list, as_of=as_of)

    expected = os.getenv("JBT_API_KEY", "")
    if not expected:

## 5. 通用问题

**文件**: 多个服务  
**示例**:
- `services/sim-trading/src/ledger/service.py` - `_ledger_instance`
- `services/decision/src/api/routes/signal.py` - `_signal_dispatcher`
- `services/backtest/src/api/routes/approval.py` - `_runner`

**问题描述**:

**影响范围**:
- 单例模式失效
- 可能导致状态不一致

import threading

_instance = None
_instance_lock = threading.Lock()

def get_instance():
    global _instance
    if _instance is None:
        with _instance_lock:
            if _instance is None:  # 双重检查
                _instance = MyClass()
    return _instance
```
