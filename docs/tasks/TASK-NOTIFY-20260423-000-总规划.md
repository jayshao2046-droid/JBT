## 静默窗口实现要求

- 全服务统一辅助函数：判断当前是否落在 **08:00–24:10**（即 `00:11–07:59` 静默）。

## 送达反馈机制要求

- 飞书 send 失败：本服务 dispatcher 自动降级邮件，邮件主题加 `[飞书失败兜底]`。
- 邮件 send 失败：写 `runtime/logs/alarm.log` 一行 JSON。
- 双通道失败：在下一次任意通道成功推送时，自动追加 `⚠️ 上次推送失败累计 N 条，最早 HH:MM:SS` 横幅。
- 每个服务必须暴露 `/api/v1/notify/health` 端点（或对应路径）：返回近 24h 飞书/邮件成功率、失败次数、最近失败原因 Top3。

### a) 静默窗口辅助函数

```python
def _in_push_window(now=None):
    """8:00-24:10 可推送；其余静默。返回 True 表示可推送。"""
    from datetime import datetime, timezone, timedelta
    CN_TZ = timezone(timedelta(hours=8))
    now = now or datetime.now(CN_TZ)
    minutes = now.hour * 60 + now.minute
    return minutes >= 8 * 60 or minutes <= 10  # 8:00-23:59 或 00:00-00:10
```

```python
TITLE_PREFIX = {

    ("alert", "P1"): "⚠️ [P1-报警]",

    ("trade", "fill"): "📈 [模拟-成交]",
    # ... (各服务在自己的 dispatcher 中定义)
}

def build_title(prefix_key, body, service_name):
    prefix = TITLE_PREFIX.get(prefix_key, "📣 [系统-通知]")
    return f"JBT {service_name} {prefix} {body}"
```

### c) 落款生成

```python
def build_footer(service_cn_name, ts=None):
    from datetime import datetime, timezone, timedelta
    CN_TZ = timezone(timedelta(hours=8))
    ts = ts or datetime.now(CN_TZ)
    return f"JBT-{service_cn_name} | {ts.strftime('%Y-%m-%d %H:%M:%S')}"
```
