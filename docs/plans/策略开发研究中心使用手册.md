## 核心工具详解

### 工具1: SymbolProfiler（品种特征分析器）

**用途**：分析期货品种的历史数据，提取关键特征用于策略生成。

**使用方式**：
```python
from services.decision.src.research.symbol_profiler import SymbolProfiler

profiler = SymbolProfiler(
    mini_api_url="http://192.168.31.74:8105",
    cache_dir="runtime/symbol_profiles"
)

# 计算品种特征（自动使用增量更新）
features = await profiler.calculate_features(
    symbol="rb",
    interval=1440,  # 日K线
    enable_cache=True
)

print(f"波动率: {features['volatility_1y']:.2%}")
print(f"趋势强度: {features['trend_strength_1y']:.2f}")
print(f"流动性评分: {features['liquidity_score']:.2f}")
```

**特征维度**（35个期货品种 × 20+维度）：
| 特征类别 | 特征名称 | 说明 |
|---------|---------|------|
| 波动率 | `volatility_3m`, `volatility_1y`, `volatility_5y` | 3个月/1年/5年年化波动率 |
| 趋势 | `trend_strength_3m`, `trend_strength_1y` | 趋势强度（-1到1） |
| 流动性 | `liquidity_score` | 流动性评分（0-100） |
| 季节性 | `seasonality_jan`, `seasonality_feb`, ... | 每月季节性系数 |
| 相关性 | `correlation_with_index` | 与指数相关性 |

**缓存机制**：
- 首次计算：获取5年历史数据（~1260条日K），耗时3-5秒
- 增量更新：只获取1天新数据（1条日K），耗时0.5秒
- 缓存命中：无需获取数据，耗时0.1秒

---

### 工具2: ThreeTierOptimizer（三层策略优化器）

**用途**：基于品种特征自动生成适配的策略配置。

**三层架构**：
1. **Tier 1 - 品种分析层**：调用 SymbolProfiler 获取品种特征
2. **Tier 2 - 策略生成层**：根据特征选择策略模板和参数
3. **Tier 3 - 参数优化层**：调用 MetaOptimizer 优化参数

**使用方式**：
```python
from services.decision.src.research.three_tier_optimizer import ThreeTierOptimizer

optimizer = ThreeTierOptimizer()

# 为单个品种生成策略
result = await optimizer.generate_strategy_for_symbol(
    symbol="rb",
    force_recalculate=False  # 使用缓存的特征
)

print(f"策略路径: {result['strategy_yaml_path']}")
print(f"策略类型: {result['strategy_type']}")
print(f"预期夏普: {result['expected_sharpe']:.2f}")
```

**策略模板选择逻辑**：
| 品种特征 | 推荐策略 | 参数调整 |
|---------|---------|---------|
| 高波动率（>30%） | 突破策略 | 加大止损倍数（ATR×2.5） |
| 低波动率（<15%） | 均值回归 | 缩小止损倍数（ATR×1.5） |
| 强趋势（>0.6） | 趋势跟踪 | 延长均线周期（MA30/60） |
| 弱趋势（<0.3） | 震荡策略 | 缩短均线周期（MA5/10） |

---

### 工具3: MetaOptimizer（参数优化器）

**用途**：对策略参数进行网格搜索或贝叶斯优化，找到最优参数组合。

**使用方式**：
```python
from services.decision.src.research.meta_optimizer import MetaOptimizer

optimizer = MetaOptimizer()

# 优化策略参数
result = await optimizer.optimize_strategy(
    symbol="rb",
    strategy_path="runtime/three_symbols_strategies/rb_adaptive_strategy.yaml",
    optimization_rounds=5,  # 优化轮次
    metric="sharpe_ratio"   # 优化目标：sharpe_ratio / total_return / profit_factor
)

print(f"原始夏普: {result['original_sharpe']:.2f}")
print(f"优化后夏普: {result['optimized_sharpe']:.2f}")
print(f"参数变化: {result['param_changes']}")
```

**优化参数范围**：
```python
# 示例：均线交叉策略参数范围
param_ranges = {
    "fast_period": [5, 10, 15, 20],      # 快线周期
    "slow_period": [20, 30, 40, 60],     # 慢线周期
    "atr_multiplier": [1.5, 2.0, 2.5],   # ATR止损倍数
    "risk_reward_ratio": [1.5, 2.0, 3.0] # 盈亏比
}
```

**优化算法**：
- **网格搜索**：遍历所有参数组合（适合参数空间小的情况）
- **贝叶斯优化**：智能采样（适合参数空间大的情况，推荐）

**用途**：将YAML策略配置转换为可执行的交易信号。

**使用方式**：
```python

print(f"信号类型: {signal['action']}")  # BUY / SELL / HOLD
print(f"入场价: {signal['entry_price']}")
print(f"止损价: {signal['stop_loss']}")
```

## 策略配置YAML格式详解

### 标准模板位置

JBT 使用 **TqSDK 标准格式**的 YAML 策略模板，确保生成的策略可以直接进入 TqSDK 回测系统：

**模板路径**：`services/decision/src/research/templates/strategy_template.yaml`

**用途**：
- CodeGenerator 在生成策略时会读取此模板
- 确保所有生成的策略都符合 TqSDK 回测格式
- 保证策略可以直接进入回测系统和 TqSDK 进行验证

**重要性**：
- ✅ 符合 TqSDK 官方回测格式
- ✅ 包含完整的风控参数（止损、止盈、熔断）
- ✅ 包含资金管理参数（仓位比例、动态调整）
- ✅ 包含交易成本参数（滑点、手续费）
- ✅ 支持因子权重配置

### 基础结构（TqSDK 标准格式）

```yaml
# ============================================================
# JBT 标准策略导入模板 v1.0
# 适用于: TqBacktest 真实回测
# ============================================================

# ──────────────────────────────────────────────────────────
# 一、基本信息
# ──────────────────────────────────────────────────────────
name: "FC-001_rb_trend_5m"                 # 策略唯一标识
description: "螺纹钢趋势跟踪策略"           # 策略说明
version: "1.0"                             # 版本号
category: "trend_following"                # 策略分类

# 策略分类选项：
# - trend_following: 趋势跟踪
# - mean_reversion: 均值回归
# - breakout: 突破策略
# - arbitrage: 套利策略
# - intraday_momentum: 日内动量
# - intraday_oscillation: 日内震荡
# - intraday_reversal: 日内反转

# ──────────────────────────────────────────────────────────
# 二、回测基础参数
# ──────────────────────────────────────────────────────────
symbols:                                   # 交易品种（必须带交易所前缀）
  - "SHFE.rb2505"                          # DCE=大商所 SHFE=上期所 CZCE=郑商所 CFFEX=中金所

timeframe_minutes: 5                       # K线周期（分钟）: 1/3/5/15/30/60/240/1440

# ──────────────────────────────────────────────────────────
# 三、因子/指标配置
# ──────────────────────────────────────────────────────────
# 可用因子（共 23 个）:
#   趋势类: SMA, EMA, MACD, ADX, DEMA, EMA_Cross, EMA_Slope, Ichimoku, ParabolicSAR, Supertrend
#   反转类: RSI, BollingerBands, KDJ, WilliamsR, CCI
#   突破类: DonchianBreakout
#   波动类: ATR, ATRTrailingStop, HistoricalVol
#   量价类: VolumeRatio, OBV, VWAP, MFI

factors:                                   # 因子列表（至少1个）
  - factor_name: "MACD"                    # 因子名称（区分大小写）
    weight: 0.4                            # 因子权重（0-1，总和建议为1.0）
    params:                                # 因子参数
      fast: 12
      slow: 26
      signal: 9

  - factor_name: "RSI"
    weight: 0.3
    params:
      period: 14

  - factor_name: "ATR"                     # 建议始终包含 ATR
    weight: 0.3
    params:
      period: 14

# ──────────────────────────────────────────────────────────
# 四、市场过滤（可选，推荐启用）
# ──────────────────────────────────────────────────────────
market_filter:                             # 进场前的前置过滤条件
  enabled: true                            # 是否启用
  conditions:
    - "atr > 0.005 * close"               # 波动率过滤（避免横盘）
    - "adx > 20"                          # 趋势强度过滤

# ──────────────────────────────────────────────────────────
# 五、信号规则
# ──────────────────────────────────────────────────────────
signal:                                    # 开仓信号逻辑
  long_condition: "macd_hist > 0 and rsi > 50 and rsi_slope > 0"
  short_condition: "macd_hist < 0 and rsi < 50 and rsi_slope < 0"
  confirm_bars: 1                          # 信号确认K线数

# ──────────────────────────────────────────────────────────
# 六、仓位管理
# ──────────────────────────────────────────────────────────
position_fraction: 0.08                    # 初始仓位比例（8%）

position_adjustment:                       # 动态仓位调整方式
  method: "atr_scaling"                   # atr_scaling = ATR 比例缩放
  base_position: 0.08
  atr_period: 14
  atr_multiplier: 1.0

# ──────────────────────────────────────────────────────────
# 七、交易成本（必填）
# ──────────────────────────────────────────────────────────
transaction_costs:                         # 必填，否则无法进入 TqSDK 回测
  slippage_per_unit: 1                    # 每手滑点（最小变动价位数）
  commission_per_lot_round_turn: 5        # 每手双边佣金（元）

# ──────────────────────────────────────────────────────────
# 八、风控参数
# ──────────────────────────────────────────────────────────
risk:
  # 系统级风控（JBT 内部处理）
  daily_loss_limit_yuan: 2000             # 当日最大亏损额（元）
  per_symbol_fuse_yuan: 160              # 单品种熔断额（元）
  max_drawdown_pct: 0.008                # 最大回撤比例限制（0.8%）
  
  # TqSDK 回测参数
  force_close_day: "14:55"              # 日盘强制平仓时间
  force_close_night: "22:55"            # 夜盘强制平仓时间（无夜盘可删除）
  no_overnight: true                     # 是否禁止持仓过夜

# 止损止盈规则（使用 ATR 倍数）
stop_loss:                                 # 止损配置
  atr_multiplier: 1.5                    # 止损距离 = ATR × 1.5
  type: "atr"                            # 类型: atr / fixed_pct / fixed_yuan

take_profit:                               # 止盈配置
  atr_multiplier: 2.5                    # 止盈距离 = ATR × 2.5
  type: "atr"

# ──────────────────────────────────────────────────────────
# 九、标签（可选）
# ──────────────────────────────────────────────────────────
tags: ["v1", "trend", "5m", "SHFE", "rb"]   # 自由标签
```

### 关键差异说明

**与传统格式的主要区别**：

| 字段 | TqSDK 格式 | 说明 |
|-----|-----------|------|
| **版本信息** | `version`, `category` | 必须包含版本号和策略分类 |
| **因子权重** | `factors[].weight` | 每个因子需要配置权重（0-1） |
| **市场过滤** | `market_filter.enabled` | 需要明确启用/禁用开关 |
| **仓位调整** | `position_adjustment` | 支持动态仓位调整（atr_scaling） |
| **止损止盈** | `stop_loss.atr_multiplier` | 使用 ATR 倍数，不是固定金额 |
| **强制平仓** | `risk.force_close_day` | 使用具体时间点（如 "14:55"） |
| **标签** | `tags` | 支持标签分类 |

**重要说明**：
- ⚠️ **不支持固定金额止损止盈**（`stop_loss_yuan`, `take_profit_yuan`），必须使用 ATR 倍数
- ⚠️ **不支持 K 线数强制平仓**（`time_to_exit_bars`），必须使用具体时间点
- ⚠️ **不支持 `initial_capital` 配置**，TqSDK 通过 `TqSim(initial_capital)` 设置
- ✅ **必须包含 `transaction_costs`**，否则无法进入 TqSDK 回测

### 支持的入场规则类型

#### 1. 均线交叉（ma_cross）
```yaml
- type: "ma_cross"
  fast_period: 10      # 快线周期
  slow_period: 30      # 慢线周期
  direction: "long"    # long=金叉做多, short=死叉做空, both=双向
```

#### 2. 波动率突破（volatility_breakout）
```yaml
- type: "volatility_breakout"
  lookback: 20         # 回溯周期
  threshold: 1.5       # 突破阈值（ATR倍数）
  direction: "both"
```

#### 3. RSI超买超卖（rsi_reversal）
```yaml
- type: "rsi_reversal"
  period: 14           # RSI周期
  overbought: 70       # 超买线
  oversold: 30         # 超卖线
  direction: "both"
```

#### 4. 布林带突破（bollinger_breakout）
```yaml
- type: "bollinger_breakout"
  period: 20           # 均线周期
  std_dev: 2.0         # 标准差倍数
  direction: "both"
```

### 支持的出场规则类型

#### 1. 追踪止损（trailing_stop）
```yaml
- type: "trailing_stop"
  atr_multiplier: 2.0  # ATR倍数
```

#### 2. 固定止损（fixed_stop）
```yaml
- type: "fixed_stop"
  stop_loss_pct: 0.02  # 止损2%
```

#### 3. 盈利目标（profit_target）
```yaml
- type: "profit_target"
  risk_reward_ratio: 2.0  # 盈亏比2:1
```

#### 4. 时间出场（time_exit）
```yaml
- type: "time_exit"
  max_holding_days: 10    # 最大持仓天数
```

## 回测指标说明

### 核心指标
| 指标 | 说明 | 优秀标准 |
|-----|------|---------|
| `total_return` | 总收益率 | >20% |
| `sharpe_ratio` | 夏普比率 | >1.5 |
| `max_drawdown` | 最大回撤 | <15% |
| `win_rate` | 胜率 | >50% |
| `profit_factor` | 盈亏比 | >1.5 |
| `total_trades` | 交易次数 | 30-100（避免过度交易） |

### 风险指标
| 指标 | 说明 | 优秀标准 |
|-----|------|---------|
| `calmar_ratio` | 卡玛比率（收益/最大回撤） | >2.0 |
| `sortino_ratio` | 索提诺比率（下行风险调整收益） | >2.0 |
| `max_consecutive_losses` | 最大连续亏损次数 | <5 |
| `avg_win` | 平均盈利 | >平均亏损×1.5 |
| `avg_loss` | 平均亏损 | <平均盈利×0.67 |

## 常见问题FAQ

### Q1: 如何选择合适的策略类型？
**A**: 根据品种特征自动选择：
- 高波动品种（如铁矿石i、焦炭j）→ 突破策略
- 低波动品种（如白糖SR、棉花CF）→ 均值回归
- 强趋势品种（如螺纹钢rb、原油sc）→ 趋势跟踪
- 弱趋势品种（如豆粕m、玉米c）→ 震荡策略

使用 `SymbolProfiler` 查看品种特征后决策。

### Q2: 回测结果很好，但实盘表现差怎么办？
**A**: 常见原因和解决方案：
1. **过拟合**：减少优化轮次，使用样本外测试
2. **滑点成本**：回测时加入滑点参数（如0.5个tick）
3. **流动性问题**：避免在非主力合约交易
4. **数据质量**：检查历史数据是否有缺失或异常

### Q3: 如何提高策略的夏普比率？
**A**: 三个方向：
1. **降低波动率**：使用波动率调整仓位（`volatility_adjusted`）
2. **提高胜率**：增加过滤条件（如趋势过滤、波动率过滤）
3. **优化盈亏比**：调整止损止盈参数（如`risk_reward_ratio: 2.0`）

### Q4: 多品种组合如何配置？
**A**: 遵循分散化原则：
```python
# 选择低相关性品种
portfolio = {
    "黑色系": ["rb", "hc", "i"],      # 相关性高，选1-2个
    "化工": ["MA", "TA", "PP"],       # 相关性高，选1-2个
    "农产品": ["m", "c", "a"],        # 相关性中等，选2-3个
    "能源": ["sc", "fu"],             # 独立品种
    "贵金属": ["au", "ag"]            # 独立品种
}
```

使用 `SymbolProfiler` 的 `correlation_matrix` 功能查看品种相关性。

### Q5: 如何加速回测速度？
**A**: 三个优化方向：
1. **使用缓存**：启用 `enable_cache=True`（首次慢，后续快）
2. **并行回测**：使用 `backtest_all_strategies.py --parallel 4`
3. **减少数据量**：缩短回测周期或使用更大的K线周期

## FAQ（常见问题）

### Q1: 生成的策略缺少回测参数怎么办？

**A**: JBT 已提供标准 YAML 模板（`services/decision/src/research/templates/strategy_template.yaml`），CodeGenerator 会自动使用此模板生成完整的策略配置。如果发现生成的策略缺少参数，请检查：

1. 模板文件是否存在
2. CodeGenerator 是否正确读取了模板
3. 手动补充缺失的参数（参考模板）

### Q2: 如何修改默认的风控参数？

**A**: 编辑模板文件 `services/decision/src/research/templates/strategy_template.yaml`，修改以下参数：

```yaml
risk:
  stop_loss_yuan: 1000              # 修改止损金额
  take_profit_yuan: 3000            # 修改止盈金额
  daily_loss_limit_yuan: 2000       # 修改日最大亏损
  max_drawdown_pct: 0.015           # 修改最大回撤
  consecutive_loss_pause_days: 1    # 修改连续亏损暂停天数
  max_position_per_day: 1           # 修改每日最大开仓次数
  time_to_exit_bars: 12             # 修改强制平仓时间
  margin_usage_percent: 15          # 修改最大保证金使用率
```

### Q3: 初始资金参数为什么不生效？

**A**: 当前版本（v2.0）中，`initial_capital` 参数在回测引擎中硬编码为50万元。虽然可以在YAML中配置，但实际回测时会使用硬编码值。未来版本将支持从YAML读取此参数。

### Q4: 策略生成后如何进入回测系统？

**A**: 生成的YAML策略会自动保存到 `runtime/three_symbols_strategies/` 目录，可以直接使用以下方式回测：

1. **YamlSignalExecutor 回测**（推荐）：
   ```python
   from services.decision.src.research.yaml_signal_executor import YamlSignalExecutor
   executor = YamlSignalExecutor()
   result = await executor.backtest_strategy("runtime/three_symbols_strategies/rb_strategy.yaml")
   ```

2. **TqSDK 回测**：将YAML转换为TqSDK格式后回测

3. **批量回测脚本**：
   ```bash
   python scripts/backtest_all_strategies.py --strategies-dir runtime/three_symbols_strategies
   ```

### Q5: 如何确保策略符合生产标准？

**A**: 使用 ThreeTierOptimizer 的迭代优化功能，它会自动验证策略是否达到生产标准：

- Sharpe ≥ 1.5
- 交易次数 ≥ 30
- 胜率 ≥ 45%
- 最大回撤 ≤ 8%
- 年化收益 ≥ 15%

如果不达标，优化器会自动调用 DeepSeek V3 和 R1 进行诊断和优化。
