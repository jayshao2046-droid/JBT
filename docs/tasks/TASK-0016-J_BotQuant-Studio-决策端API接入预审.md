- **E API 接入 J_BotQuant Studio 决策端必须新建独立任务 `TASK-0016`，不得直接复用 `TASK-0012`。**

1. `TASK-0012` 的任务边界是“legacy 兼容桥接层”，目标是兼容 legacy 信号入口。

- **当前轮次归属：跨治理层任务。**

## 五、正式治理冻结

### 1. 与 `TASK-0012` 的边界

1. `TASK-0012` 只承接 legacy 兼容桥接。
2. `TASK-0016` 承接正式 Studio API 接入、测试闭环、切换门槛与回滚要求。
3. `TASK-0016` 不自动继承 `TASK-0012` 的白名单或 Token。

### 2. 未来实施拆批原则

1. 若新增或修改正式 API 契约到 `shared/contracts/**`：单独 P0 批次。
2. 若在 `services/sim-trading/**` 增加正式接入入口、校验逻辑或只读状态接口：单独 P1 批次。
3. 若在 `integrations/legacy-botquant/**` 增加外部适配逻辑：单独 P0 批次。
4. 任一批次文件范围变化，都必须重新补充预审。

### 3. 正式接入口径

1. 正式 API 接入必须保留鉴权、签名、幂等、trace、风险摘要与失败告警。
2. 正式接入不得绕过 `TASK-0009` 的风控门槛与 `TASK-0013` 的 stage preset 治理。
3. 切换前必须具备最小回滚路径，禁止“接上再说”。
