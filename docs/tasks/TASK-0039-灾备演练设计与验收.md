## 一、任务目标

在 Phase B 全闭环基础上，设计并执行 4 个灾备演练场景（DR1~DR4），验证已部署服务在真实故障场景下的韧性，确保告警及时、服务自愈、数据完整。

本轮明确目标：

1. **DR1 — Mini 断网**：拔网线或防火墙模拟断网，验证 data/sim-trading 自动重连 + 飞书 P1 告警 ≤ 30s
2. **DR2 — Studio 宕机**：docker stop 全容器，验证 restart policy 自动恢复 + 飞书 P0 告警 ≤ 30s
3. **DR3 — Docker 容器崩溃**：kill -9 主进程，验证 restart: unless-stopped 策略生效，服务 ≤ 60s 恢复
4. **DR4 — 数据端采集中断+恢复**：模拟采集源断开后恢复，验证自动补采缺失区间 + health_check 不误报

## 四、验收标准

### DR1 — Mini 断网

| # | 验收项 | 判定方式 |
|---|--------|---------|
| 1 | 飞书 P1 告警 ≤ 30s | 记录告警时间戳 vs 断网时间戳 |
| 2 | data 服务自动重连 | docker logs 确认 reconnect 成功 |
| 3 | sim-trading 自动重连 | docker logs 确认 CTP 重连 |
| 4 | bars API 恢复后数据连续 | 对比断网前后 K 线无缺口 |

### DR2 — Studio 宕机

| # | 验收项 | 判定方式 |
|---|--------|---------|
| 1 | 飞书 P0 告警 ≤ 30s | 记录告警时间戳 |
| 2 | docker restart policy 自动恢复 | docker ps 确认容器重启 |
| 3 | 决策+看板自动重连 | 页面刷新后功能正常 |

### DR3 — Docker 容器崩溃

| # | 验收项 | 判定方式 |
|---|--------|---------|
| 1 | restart: unless-stopped 策略生效 | kill -9 后 docker ps 确认重启 |
| 2 | 服务 ≤ 60s 内恢复响应 | curl 健康检查 endpoint |

### DR4 — 数据端采集中断+恢复

| # | 验收项 | 判定方式 |
|---|--------|---------|
| 1 | 恢复后自动补采缺失区间 | 检查数据时间戳连续性 |
| 2 | health_check 不误报 | 健康检查 API 返回正确状态 |
