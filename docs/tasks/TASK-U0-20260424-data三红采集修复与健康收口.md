## 任务背景

2026-04-24 Mini 健康通知持续出现 3 个红灯：

1. 持仓日报 `position_daily`
2. 外汇日线 `forex`
3. 期权行情 `options`

只读排查确认这不是单纯展示误报，而是 3 条真实采集链路都存在运行态问题；同时这 3 类数据已被 researcher、context API 和 preread 继续消费，不能通过关闭采集或消费降级处理。

## 根因确认

### 1. 持仓日报

- `PositionCollector.collect()` 在未显式指定 `trade_date` 时默认只抓当天。
- 交易日白天运行时，Tushare 当天持仓数据尚未落出，直接返回空，导致 `position_daily` 长期失败。

### 2. 期权行情

- `OptionsCollector.collect()` 同样默认只抓当天。
- `job_options()` 还会把 `today` 强行传入 pipeline，导致即使上一交易日已有可用数据，也不会回退到最近有效交易日。

### 3. 外汇日线

- `ForexCollector` 传给 `pro.fx_daily()` 的是 `USDCNH`、`EURUSD` 这类裸代码。
- Tushare `fx_daily` 实际要求 `USDCNH.FXCM`、`EURUSD.FXCM` 一类 `ts_code`，因此采集一直空跑。

### 4. 手动转绿的运行态事实

- Mini data API 进程真实环境变量为 `DATA_STORAGE_ROOT=/data`。
- `dashboard/collectors` 最终读取的是 `/data/logs/collector_status_latest.json` 与 `/data/**` 下的 freshness。
- 手动补采如果写到 `/app/runtime/data`，不会改变用户看到的 collectors 状态；要让面板转绿，补采与 snapshot 刷新必须对齐 `/data`。

## 修复摘要

### 持仓日报

- 新增最近 5 个自然日候选 `trade_date` 回退逻辑。
- 当当天无数据时，自动回退到最近可用交易日，并保留明确日志。

### 期权行情

- 新增最近 5 个自然日候选 `trade_date` 回退逻辑。
- 调度器不再向 `run_options_pipeline()` 强塞当天日期，允许 collector 自行回退到最近有效交易日。

### 外汇日线

- 以 `DEFAULT_PAIR_CODES` 统一维护别名 -> `FXCM ts_code` 映射。
- `collect()` 和 `_fetch_live()` 全链路改为按 `ts_code` 调用，并保留原始货币对标签给下游消费。
