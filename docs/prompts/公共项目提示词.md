## 固定端口 / 网络访问规则（冻结）

- 自本条生效起，前台 / 看板类 Web 端口固定为：backtest-web = 3001、sim-trading-web = 3002、decision-web = 3003、data-web = 3004、dashboard-web = 3005、live-trading-web = 3006。
- API 服务端口继续沿用既有 registry 语义，且不得与上述 Web 端口混用：sim-trading api = 8101、live-trading api = 8102、backtest api = 8103、decision api = 8104、data api = 8105、dashboard api = 8106。
- 所有设备、所有部署、所有网络路径必须统一遵守“端口固定，只有主机名 / IP 变化”的规则；本地内网、Tailscale、向日葵三类访问方式必须共用同一端口语义，禁止按网络入口分别改端口。

## 9. JBT 统一通知卡片标准（全服务强制执行）

> 本标准由 Jay.S 于 2026-04-07 确认，适用于 JBT 所有服务（sim-trading / live-trading / backtest / decision / data / dashboard）的全部通知渠道（飞书 Webhook + Email SMTP）。后续任何新增通知实现必须严格遵守，不得自定义颜色或结构。

### 颜色映射（7类）

| 通知类型 | 飞书 `template` | 邮件 header HEX | 图标 | 典型触发场景 |
|---------|--------------|----------------|------|------------|
| 报警 P0 | `red` | `#c0392b` | 🚨 | 系统崩溃、强制平仓、数据丢失 |
| 报警 P1 | `orange` | `#e67e22` | ⚠️ | CTP 断线、认证/登录失败、交易暂停 |
| 报警 P2 | `yellow` | `#f39c12` | 🔔 | 低优先级风险、阈值预警 |
| 交易 | `grey` | `#7f8c8d` | 📊 | 成交回报、订单状态、持仓变动 |
| 资讯 | `blue` | `#2980b9` | 📈 | 市场行情分析、策略资讯、日报 |
| 新闻 | `wathet` | `#5dade2` | 📰 | 财经新闻推送、宏观资讯 |
| 通知 | `turquoise` | `#1abc9c` | 📣 | 系统服务启停、定时报表、心跳 |

### 飞书卡片结构（标准模板）

```
msg_type: "interactive"
card:
  header:
    title: "{icon} [{STAGE}-{TYPE}] {标题}"
    template: {对应 template 值}
  elements:
    1. tag: div (lark_md) → 核心信息（因类型不同内容不同）
    2. tag: hr
    3. tag: div (lark_md) → 追踪/次要信息
    4. tag: hr
    5. tag: note → "JBT {service_name} | YYYY-MM-DD HH:MM:SS"
```

### 邮件 HTML 结构（标准模板）

```html
<div class="card">
  <div class="header" style="background:{HEX色值}; color:#fff; padding:16px 20px;">
    <h2>{icon} [{STAGE}-{TYPE}] {标题}</h2>
    <div class="sub">JBT {service_name} 通知</div>
  </div>
  <div class="body" style="padding:16px 20px;">
    <p class="section-title">核心信息</p>
    <table><!-- 原因/账户/事件/品种 --></table>
    <hr>
    <p class="section-title">追踪信息</p>
    <table><!-- 任务/环境/级别/策略/信号/追踪ID --></table>
  </div>
  <div class="footer" style="font-size:11px; color:#aaa; padding:10px 20px;">
    JBT {service_name} | {timestamp}
  </div>
</div>
```

### 强制规则

1. **颜色不得自定义**：所有服务发送通知必须严格按上表选色。
2. **飞书禁用纯文本**：必须使用 `msg_type: "interactive"` + 卡片结构，禁止 `msg_type: "text"`。
3. **邮件禁用纯文本**：必须发送 HTML Card 格式。
4. **标题格式固定**：`{icon} [{STAGE}-{TYPE}] {标题}`，不得省略图标和分类标签。
5. **底部时间戳必须**：飞书 note / 邮件 footer 必须包含服务名和时间戳。
6. **标签命名规则**：报警类用 `P0/P1/P2`；非报警类用 `TRADE/INFO/NEWS/NOTIFY`。
7. **已有实现参考**：`services/sim-trading/src/notifier/feishu.py` 与 `email.py`（commit `a014f13`）为当前基准实现，其他服务复制该模板并按类型替换颜色。
