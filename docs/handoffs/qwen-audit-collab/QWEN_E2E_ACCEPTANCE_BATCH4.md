# QWEN_E2E_ACCEPTANCE_BATCH4.md

## §1 任务摘要

本文件定义Batch-4的端到端验收场景，验证CA2'期货沙箱回测引擎、CB2'股票沙箱回测引擎、CG3回测端股票手动回测与看板调整、CA6信号真闭环→sim-trading接口等功能的完整实现和集成效果。同时提供Phase C的完整进度快照。

## §2 批量验收范围

### 2.1 功能验收范围
- CA2' 期货沙箱回测引擎：验证内部快速回测能力
- CB2' 股票沙箱回测引擎：验证内部快速回测能力（股票特化）
- CG3 股票手动回测与看板调整：验证人工审核流程和看板展示
- CA6 信号真闭环→sim-trading接口：验证信号传递链路

### 2.2 非功能验收范围
- 性能指标：响应时间、并发处理能力
- 可靠性：错误处理、重试机制
- 安全性：认证授权、数据保护
- 可扩展性：模块化设计、接口兼容性

## §3 端到端验收场景

### 3.1 期货策略完整生命周期场景
**场景描述**：验证期货策略从研发到实盘的完整生命周期
1. 研究人员在决策服务中编写期货策略
2. 使用CA2'期货沙箱进行快速参数优化
3. 将策略提交到外部回测服务进行深度验证
4. 通过CA6接口将信号发送到sim-trading执行
5. 监控策略表现并根据结果调整参数

**验收标准**：
- 沙箱回测响应时间 ≤ 500ms
- 沙箱与外部回测结果偏差 ≤ 0.1%
- 信号传递延迟 ≤ 100ms
- 策略执行准确率 100%

### 3.2 股票策略完整生命周期场景
**场景描述**：验证股票策略从研发到实盘的完整生命周期
1. 研究人员在决策服务中编写股票策略
2. 使用CB2'股票沙箱进行快速参数优化
3. 通过CG3手动回测流程进行人工审核
4. 审核通过后通过CA6接口发送信号到sim-trading
5. 监控策略表现并根据结果调整参数

**验收标准**：
- 沙箱回测响应时间 ≤ 300ms
- 正确处理T+1交易制度
- 正确处理涨跌停限制
- 人工审核流程顺畅
- 信号传递延迟 ≤ 100ms

### 3.3 沙箱性能压力场景
**场景描述**：验证沙箱引擎在高并发下的性能表现
1. 同时发起50个期货沙箱回测请求
2. 同时发起50个股票沙箱回测请求
3. 监控系统资源使用情况
4. 验证回测结果准确性

**验收标准**：
- 95%的请求响应时间 ≤ 1s
- 系统CPU使用率 < 80%
- 内存使用稳定无泄漏
- 所有回测结果准确

### 3.4 信号传递可靠性场景
**场景描述**：验证信号在各种网络条件下可靠传递
1. 正常网络条件下信号传递测试
2. 网络延迟较高时的信号传递测试
3. sim-trading服务短暂不可用时的重试机制
4. 信号状态查询功能验证

**验收标准**：
- 正常条件下信号传递成功率 100%
- 网络延迟时信号传递成功率 > 99%
- 服务不可用时自动重试成功
- 状态查询准确反映实际状态

### 3.5 人工审核流程场景
**场景描述**：验证股票策略的人工审核流程
1. 提交股票策略进行手动回测
2. 审核人员查看策略详情和风险指标
3. 批准或拒绝策略
4. 验证批准后的执行和拒绝后的处理

**验收标准**：
- 审核流程清晰易用
- 风险指标展示准确
- 审核决策记录完整
- 批准后能正常执行
