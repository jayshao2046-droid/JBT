## 问题描述

`services/data/src/main.py` 的 `_parse_symbol_request()` 函数对品种代码统一执行 `.lower()`，导致构建的连续合约路径为 `KQ_m_CZCE_sr`（小写），而 Docker 容器内（Linux，区分大小写）采集目录为 `KQ_m_CZCE_SR`（大写），文件系统无法匹配，所有 CZCE 品种连续合约查询返回 404。

**影响范围**：
- 已有 CZCE 品种：MA / TA / CF / OI（共 96 个策略无法取数）
- 新增 CZCE 品种：SR / FG / SA / RM / PF / UR
- DCE / SHFE 品种：不受影响（目录本身小写）

## 修复方案

在 `_resolve_symbol_frame()` 函数中，连续合约目录查找失败时，自动尝试品种代码大小写翻转（fallback），8 行改动，零破坏性。

**不修改** `_parse_symbol_request()`，保持现有解析行为，降低影响范围。

## 修复内容

- **文件**：`services/data/src/main.py`
- **函数**：`_resolve_symbol_frame()`
- **改动**：lowercase 目录不存在时，`rsplit("_", 1)` 取品种代码，翻转大小写后尝试 `alt_dir`

## 部署步骤

1. MacBook → Mini：`scp main.py jaybot@192.168.31.74:~/JBT/services/data/src/main.py`
2. Mini → 容器：`docker cp ~/JBT/services/data/src/main.py JBT-DATA-8105:/app/src/main.py`
3. 重启容器：`docker restart JBT-DATA-8105`
