## SimNow 严格风控总纲

### 1. 总原则

1. SimNow 不是“宽松沙盒”，而是“无真实资金结算的实盘风控演练环境”。
2. 风控优先级高于信号优先级。任何信号只要与风控冲突，默认拒绝执行。
3. 失败默认收紧，不默认放行。系统降级时只能进入“拒绝新开仓”或“只减仓”模式，不能静默继续交易。
4. 每一笔开仓都必须绑定明确的退出条件；没有止损、止盈、超时退出、强平时间的开仓请求一律拒绝。
5. 订单、持仓、账户、策略、会话、系统六层风控必须同时存在，不能只做单层止损。
6. 风控决策必须可追溯：谁发的信号、为什么被放行或拒绝、在哪一层触发熔断、何时恢复，都必须写入账本和告警流。

### 2. 仓前风控门闸

1. 合约白名单：只允许预先登记的交易所、品种、合约进入执行；未登记合约直接拒单。
2. 交易时段校验：不在允许交易时段、集合竞价特殊窗口、临近强平窗口时，禁止新开仓。
3. 行情新鲜度校验：行情超时、时间戳倒退、价格缺口异常、涨跌停附近不可成交时，禁止新开仓。
4. 信号有效期校验：超过 TTL 的信号直接丢弃，不允许补发执行。
5. 信号幂等校验：同一 `signal_id`、同一策略版本、同一方向的重复信号只能处理一次。
6. 版本一致性校验：信号中的策略 ID、版本号、参数摘要、风险摘要必须与当前已激活策略一致；不一致直接拒绝。
7. 账户健康校验：资金、权益、保证金占用、当日盈亏、累计回撤、熔断状态任一超限，禁止新开仓。
8. 仓位上限校验：单笔、单合约、单策略、单账户的开仓数量、名义风险、净敞口、方向集中度必须全部在限额内。
9. 价格合理性校验：下单价格必须满足最小跳动、偏离最新价上限、盘口滑点上限、追价次数上限。
10. 订单频率校验：单位时间内报单数、撤单数、改单数超过阈值时，自动进入节流或熔断。

### 3. 订单级风控

1. 同一时刻每个合约只允许有限个活动订单；超限时拒绝新增订单。
2. 新开仓订单必须在下单前计算“最坏情况下单笔风险”，超出阈值直接拒单。
3. 报单后必须等待明确回报；超时未确认则进入撤单与对账流程，禁止继续追单。
4. 连续拒单、连续撤单失败、连续回报异常达到阈值时，立刻抬升为策略级或系统级熔断。
5. 任何新开仓订单未附带保护性止损方案时，不允许发送到网关。

### 4. 持仓级风控

1. 每个持仓都必须同时存在策略止损和系统灾难止损，两者取更严格者。
2. 策略止损可以来自固定百分比、ATR、结构位、时间止损，但不能缺失。
3. 系统灾难止损独立于策略逻辑存在；即使策略失效、指标异常、信号中断，系统止损仍必须可执行。
4. 支持分级止盈，但止盈不能替代止损。
5. 支持移动止损、保本止损、时间止损；一旦触发，优先执行减仓或平仓。
6. 到达会话强平时间时，无论策略是否还看多或看空，都必须按规则执行减仓或平仓。
7. 若市场跳空直接越过止损价，必须按“最先可成交价格”执行退出，并记录滑点异常事件。

### 5. 策略级风控

1. 单策略当日累计亏损达到阈值时，立即禁止该策略继续新开仓。
2. 单策略连续亏损笔数达到阈值时，进入冷静期；冷静期内只允许平仓，不允许新开仓。
3. 单策略当日触发止损次数达到阈值时，视为策略失稳，进入策略熔断。
4. 单策略的最大净持仓、最大名义敞口、最大同向加仓次数、最大开仓频率必须独立配置。
5. 策略参数或版本一旦变更，未重新审核前不得直接复用历史风控阈值。

### 6. 账户级风控

1. 账户层必须跟踪实时权益、可用资金、保证金占用率、当日已实现盈亏、当日未实现盈亏、当日净值回撤。
2. 账户保证金占用率过高时，先进入“只减仓”，再进入“全停新单”。
3. 账户当日累计亏损、账户当日净值回撤、账户连续异常报单达到阈值时，进入账户熔断。
4. 账户熔断后必须自动撤销全部可撤新开仓挂单，并禁止新开仓；是否保留已有减仓指令由风险级别决定。
5. 账户级熔断未解除前，任何单策略都不得绕过恢复交易。

### 7. 会话级与系统级风控

1. 会话级风控负责交易日切换、日盘夜盘窗口、收盘前强平、节假日前平仓、跨会话状态锁定。
2. 系统级风控负责网关连接、行情连接、决策心跳、账本写入、时钟漂移、配置漂移、告警通道等基础健康。
3. 任一核心链路失联时，系统至少进入“拒绝新开仓”；若失联影响订单状态确认，则升级为“只减仓”或“全停”。
4. 账本、订单回报、持仓快照三者不一致时，立即进入系统熔断，先对账后恢复。
5. 任何无法确认真实持仓和订单状态的场景，都不能继续开新仓。

### 8. API 与桥接链路风控

1. legacy 到 JBT 的信号桥接必须有鉴权，不允许匿名或裸接口推送。
2. 每条信号必须带唯一 ID、来源、策略版本、产生时间、过期时间、风险摘要、请求签名。
3. 桥接层必须校验重复信号、乱序信号、过期信号、未来时间信号。
4. 桥接层只负责转换和投递，不得在桥接层自行生成交易决策。
5. 桥接失败、重试超限、鉴权失败、签名失败、字段缺失、版本不兼容时，必须拒绝并告警，不得静默吞掉。

### 9. 监控与告警

1. 风险事件至少分为 P0、P1、P2 三档。
2. P0：账户熔断、系统熔断、持仓不一致、订单状态不确定、灾难止损失效、强平失败。
3. P1：策略熔断、合约熔断、连续拒单、连续止损、行情长时间陈旧、桥接失败。
4. P2：风控阈值逼近、延迟升高、单次撤单超时、轻微滑点超标。
5. 所有告警都必须可落日志、可追溯、可关联到策略、账户、合约、订单和信号 ID。

### 10. 熔断矩阵

| 层级 | 典型触发条件 | 自动动作 | 恢复条件 |
|---|---|---|---|
| 单笔 | 单笔最坏风险超限、价格偏离超限、信号过期 | 拒绝该笔订单 | 新信号重新评估 |
| 合约 | 单合约当日亏损超限、单合约止损次数超限、行情失真 | 该合约只减仓或全停 | 行情恢复、人工复核、冷静期结束 |
| 策略 | 单策略日亏损超限、连续亏损超限、连续拒单超限 | 关闭该策略新开仓，撤销其新开仓挂单 | 原因确认、策略复核、Jay.S 明确恢复 |
| 账户 | 账户日亏损超限、保证金占用超限、净值回撤超限 | 全账户只减仓或全停，撤可撤挂单 | 对账完成、健康窗口恢复、Jay.S 明确恢复 |
| 会话 | 收盘前强平窗口、节假日前锁仓规则、跨会话风险锁定 | 强制减仓/平仓，禁止跨会话乱留仓 | 进入下一允许会话并完成复核 |
| 系统 | 网关断连、账本异常、持仓不一致、心跳中断 | 系统熔断，禁止新单，必要时只减仓 | 链路恢复、状态一致、人工审核通过 |

### 11. 默认建议阈值基线

1. 单笔最坏风险不超过账户权益的 `0.5%`。
2. 单合约当日累计亏损不超过账户权益的 `1.0%`。
3. 单策略当日累计亏损不超过账户权益的 `1.5%`。
4. 账户当日累计亏损达到账户权益的 `2.0%` 时，触发账户熔断。
5. 账户净值回撤达到 `2.5%` 进入只减仓，达到 `3.0%` 进入全停。
6. 保证金占用率达到 `60%` 告警，达到 `70%` 只减仓，达到 `80%` 全停。
7. 连续亏损 `3` 笔进入冷静期，连续亏损 `5` 笔触发策略熔断。
8. 行情陈旧超过 `2` 秒禁止新开仓，超过 `5` 秒触发系统级告警或熔断。
9. 信号 TTL 默认不超过 `3` 秒；超过即丢弃。
10. 报单确认超时默认 `2` 秒进入撤单与对账；撤单确认超时默认 `3` 秒升级告警。

说明：以上为默认建议基线。若品种、会话、策略频率或账户规模不同，项目架构师与模拟交易 Agent 必须在正式建档时把阈值写成配置与契约，不允许留成“后续再说”。

### 12. 解除熔断规则

1. 不允许“重启服务后自动恢复交易”。
2. 解除熔断前必须满足：原因确认、订单状态清楚、持仓状态清楚、账本对齐、行情与网关恢复健康、告警已送达。
3. 策略级和账户级熔断恢复，必须经 Jay.S 明确确认后才能重新开仓。
4. 系统级熔断恢复前，项目架构师必须给出“风险已收口”的简明结论。
5. 恢复交易时应优先从“只减仓”回到“允许新开仓”，不允许一步从系统熔断直接跳回全量放开。
