## 四、历史数据丢失分析 ⚠️

### 4.1 根本原因
**Docker volume 在容器重建时被清空**

容器使用独立的 Docker volume (`jbt_jbt-data-storage`)，而非 bind mount:
```json
{
  "Type": "volume",
  "Name": "jbt_jbt-data-storage",
  "Source": "/var/lib/docker/volumes/jbt_jbt-data-storage/_data",
  "Destination": "/data"
}
```

### 4.2 数据丢失时间线
1. **2026-04-12 之前**: 容器正常运行，累积 42006 个文件（2.3GB）
2. **2026-04-12 21:10**: 容器重建（修复 Dockerfile 路径问题）
3. **2026-04-12 21:12**: 新容器启动，volume 被清空
4. **2026-04-12 21:52**: 开始新数据采集（仅轻量级任务）
