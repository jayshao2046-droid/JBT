## 四、正式治理冻结

### 1. 同一风控核心 + 不同执行端适配层

1. 风控核心只能依赖统一的风险输入、统一状态、统一规则与统一输出命令，不得直接依赖 SimNow 或 live-trading 的专属网关调用细节。
2. SimNow / 实盘的差异，只允许存在于执行端适配层，包括：网关接入、账户回报格式、订单回报格式、会话环境差异与运行态注入方式。
3. `services/sim-trading/**` 与 `services/live-trading/**` 允许分别拥有自己的 gateway adapter，但不允许各自长期维护一套不同的风控规则内核。
4. 若后续需要共享可执行风控核心，唯一允许的共享代码目录只能是 `shared/python-common/**`；禁止跨服务直接 import 对方业务模块。

### 2. 阶段预设（stage preset）

1. 风控参数至少必须支持 `sim` 与 `live` 两类 preset。
2. preset 至少应覆盖：阈值基线、会话限制、只减仓 / 全停策略、通知路由、恢复门槛、部署前验证开关。
3. 切换执行阶段只能通过配置 / preset 完成，不允许靠改代码分支、注释规则或隐藏环境变量来切换。
4. preset 缺失、字段不完整或加载失败时，系统必须回落到更严格的 Fail-Safe 模式，不得默认放宽。

### 3. 正式可迁移口径

1. 风控层级冻结为：L1 事前预防、L2 事中监控、L3 事后审计。
2. Fail-Safe 冻结为：异常时只能进入拒绝新开仓、只减仓、全停、强平等风险收缩模式。
3. 独立通知冻结为：飞书 + 邮件双通道；通知链失败必须反向抬升系统风险状态。
4. 远端部署冻结为：Mini 部署与切换前验证是正式验收项，不是部署后的补做项。
5. Mini 部署前验证除健康检查、只减仓、独立通知、上游接入路径与回滚路径外，还必须覆盖“断网 / 断数据源下的本地缓存行为验证”；该项属于系统级风控执行验证细节，不构成新增规则。
6. 上述验证的目标行为冻结为：Studio 侧 L1 / L2 决策在 Mini 断网或上游数据源中断时，要么正确读取最近一次本地快照继续做安全降级判断，要么正确拒绝开仓并进入 Fail-Safe；两条路径都不得 crash，也不得产出错误交易信号。
7. 旧风控参考方案只允许被转写为上述 JBT 正式口径，不允许原文照搬或整段复制。
