# backtest

## 角色

JBT 回测服务，负责在线回测执行、绩效评估，以及回测通过后的策略推送。

## 服务方向

1. **在线回测主路径**：基于 TqSdk `TqApi(TqSim(), backtest=TqBacktest(...), auth=TqAuth(...))` 驱动的在线期货回测，不依赖本地历史数据文件。
2. **固定模板 + 用户参数**：首个可运行策略采用固定策略模板，并由用户上传一体化 YAML 文件承载策略参数与风控参数。
3. **独立回测看板**：独立前端看板（`V0-backtext 看板/`）服务于在线回测结果展示，不依赖本地数据文件。
4. **同机同 compose 部署**：回测后端与看板容器部署在同一独立设备，并由同一 docker-compose 编排（D-BT-05）。
5. **回测后推送决策端**：只有通过回测审核的策略，才允许经推送接口流向决策端（D-BT-04）。

## 固定约束

1. 回测必须是在线回测，不发起实时下单，不持有真实交易主账本。
2. 策略参数与风控参数统一来自用户上传的一体化 YAML 文件，所有风控字段必须从 YAML 读取，禁止硬编码（D-BT-03，Q-NEW）。
3. 第一阶段只支持期货回测（D-BT-01）；股票适配器接口保留但不启用。
4. 回测结果通过 API 提供给 decision 或 dashboard，策略只有在回测端通过审核后才允许推送至 decision。

## 目录职责

- `src/`: 回测执行引擎、REST API、结果汇总
- `tests/`: 回测服务测试
- `configs/`: 回测参数、风控规则配置
- `V0-backtext 看板/`: 独立回测看板原型（Next.js）
