# TASK-P1-20260424F5 — qwen3:14b 金融领域知识增强（与 F4 合并训练）

## 任务类型
- P1 标准流程
- 服务归属：跨设备工程任务（Studio 训练 + Alienware 部署）
- 母任务：与 TASK-P1-20260424F4 并行同源（**同一次 LoRA 训练同时承载 JSON 固化 + 金融知识增强两个目标**）
- 当前状态：可进入 Jay.S 文件级 Token 签发与实施
- 执行模式：Studio 24h 持续训练，与 F4 共用同一份合并数据集

## 背景
F4 解决"输出格式稳定（JSON / no_think / 品种命中）"问题，但 qwen3:14b 通用模型在以下金融纵深领域仍偏弱，是后续工作的真实瓶颈：

1. **期货合约规则**：主力换月规则、保证金阶梯、涨跌停板、限仓制度、交割流程
2. **宏观因子语义**：CFTC 持仓报告解读、FOMC 措辞、ECB 鹰鸽派、PMI/CPI/PPI 联动
3. **商品基本面**：库存-基差-月差-跨品种价差的逻辑链；天气-种植-收割对农产品的传导
4. **技术分析口径**：与 JBT 内部因子库（动量/波动率/趋势/成交量）一致的术语
5. **风控语境**：drawdown / Sharpe / Sortino / Calmar / VaR / 凯利公式 的中文语境理解

这些不靠 RAG 临时塞 prompt 能稳定解决，必须通过领域 LoRA 把"先验知识"写进权重。

## 目标
1. 在 F4 同一次 LoRA 训练中，把金融领域知识增强语料一并喂入，**不另开训练**。
2. 输出新模型 `qwen3-jbt-news:14b-q4_K_M` 同时具备：
   - F4 目标：JSON 稳定、no_think、品种命中
   - F5 目标：上述 5 类金融领域问答能力提升
3. 在 100 道金融领域 holdout 题目上，正确率从基线 ~60% 提升到 ≥ 80%。
