# JBT 量化控制台 - 统一门户 UI 设计任务

## 项目背景

我有一个量化交易系统，包含 4 个独立的看板端（模拟交易、回测、数据、决策），现在需要合并成一个统一的网站，并新增登录页、首页和系统设置页。

所有页面已经用 Next.js 15 + React 19 + shadcn/ui 开发完成，风格统一（深色主题，黑色背景）。

### 任务 4：微调四端底色（保持协调）

**要求**：
- 当前所有页面都是纯黑色背景（`bg-black`）
- 需要为四端分别设置微调的底色，但整体风格保持协调
- 建议方案：
  - **模拟交易**：`bg-neutral-950`（偏冷色调，强调风控）
  - **回测**：`bg-zinc-950`（偏中性，强调分析）
  - **数据**：`bg-slate-950`（偏蓝色调，强调数据）
  - **决策**：`bg-stone-950`（偏暖色调，强调决策）
  - **首页/登录/设置**：保持 `bg-black`（纯黑，强调统一）
- 调整方式：
  - 修改每个端的 `app/page.tsx` 根元素的 `className`
  - 确保卡片、边框、文字颜色与新底色协调
  - 保持橙色主题色不变

**参考**：
- 当前颜色变量见 `sim-trading/app/globals.css`

### 任务 5：统一所有开关、KPI、动画

**要求**：

#### 开关组件统一
- 使用 shadcn/ui 的 `Switch` 组件
- 统一样式：
  - 开启状态：橙色（`bg-orange-500`）
  - 关闭状态：灰色（`bg-neutral-700`）
  - 过渡动画：`transition-all duration-200`
- 统一尺寸：默认尺寸（不要过大或过小）

#### KPI 卡片统一
- 统一布局：
  - 标题（左上）
  - 数值（中间，大字体）
  - 变化趋势（右上，带箭头图标）
  - 副标题/说明（底部，小字体）
- 统一颜色：
  - 正值/上涨：绿色（`text-green-500`）
  - 负值/下跌：红色（`text-red-500`）
  - 中性：白色（`text-white`）
- 统一动画：
  - 数值变化时：`transition-all duration-300`
  - Hover 效果：`hover:scale-105 transition-transform`

#### 图表统一
- 使用 recharts 库
- 统一配色：
  - 线条颜色：橙色（`#f97316`）
  - 网格颜色：深灰色（`#262626`）
  - 文字颜色：浅灰色（`#a3a3a3`）
- 统一动画：
  - 进入动画：`animationDuration={300}`
  - Tooltip 动画：`isAnimationActive={true}`

#### 按钮统一
- 主按钮：橙色背景（`bg-orange-500 hover:bg-orange-600`）
- 次按钮：透明背景（`bg-transparent border border-neutral-700 hover:bg-neutral-800`）
- 危险按钮：红色背景（`bg-red-500 hover:bg-red-600`）
- 统一圆角：`rounded-md`
- 统一过渡：`transition-all duration-200`

#### 加载动画统一
- 使用 lucide-react 的 `RefreshCw` 图标
- 旋转动画：`animate-spin`
- 颜色：橙色（`text-orange-500`）

#### Toast 通知统一
- 使用 sonner 库
- 统一样式：深色主题
- 统一位置：右上角（`position: "top-right"`）
- 统一持续时间：3 秒

**参考**：
- 开关示例见 `sim-trading/app/page.tsx`
- KPI 示例见 `sim-trading/app/operations/page.tsx`
- 图表示例见 `backtest/app/page.tsx`
