## 二、失败原因详解

### 2.1 策略生成失败（2/7）

**错误信息：**
```
[ERROR] pipeline_35:   ✗ 生成失败 (p_trend_60m_v1): 'recommended_factors'
[ERROR] pipeline_35:   ✗ 生成失败 (p_trend_15m_v1): 'recommended_factors'
```

**根本原因：**
- 文件：`services/decision/src/research/code_generator.py:177`
- 代码直接访问 `strategy_design['recommended_factors']`，未做容错处理
- LLM 返回的策略设计 JSON 中缺少 `recommended_factors` 字段
- 导致 KeyError 异常

**影响范围：**
- 2 个趋势类策略未能生成
- 无重试机制，直接跳过

**已修复：**
```python
# 修复前
factors_desc = "\n".join([
    f"- {factor}: 需要配置合理的参数"
    for factor in strategy_design['recommended_factors']
])

# 修复后
recommended_factors = strategy_design.get('recommended_factors', [])
factors_desc = "\n".join([
    f"- {factor}: 需要配置合理的参数"
    for factor in recommended_factors
]) if recommended_factors else "- 根据策略逻辑自行选择合适的技术因子"
```

**修复状态：** ✅ 已完成（下次执行生效）

### 2.2 TqSdk 回测失败（5/5）

**错误信息：**
```
httpx.HTTPStatusError: Client error '422 Unprocessable Entity' for url 'http://192.168.31.142:8103/api/backtest/run'

{"detail":"正式回测不接受 runtime params 覆盖。请先修改 YAML 并重新导入，系统不会在执行前静默改写策略参数。"}
```

**根本原因：**
- 文件：`services/decision/src/research/tqsdk_backtest_client.py:112-121`
- 当前实现：Optuna 调优后，通过 API 的 `params` 字段传递最优参数
- TqSdk 服务策略：拒绝 runtime 参数覆盖，要求修改 YAML 后重新导入
- 设计冲突：流水线期望动态传参，TqSdk 要求静态配置

**当前代码逻辑：**
```python
payload = {
    "strategy_id": "generic",
    "strategy_name": strategy_name,
    "engine_type": "tqsdk",
    "start": start_date,
    "end": end_date,
    "symbols": symbols,
    "initial_capital": initial_capital,
    "params": strategy,  # ❌ TqSdk 拒绝这种覆盖
}
```

**需要修复的流程：**
1. Optuna 调优完成后，获取最优参数
2. **修改 YAML 文件**，将最优参数写入 `factors` 配置
3. 重新调用 `/api/strategy/import` 导入更新后的 YAML
4. 提交回测时不传 `params` 字段

**影响范围：**
- 所有策略的 TqSdk 回测环节全部失败
- 无法获取 TqSdk 回测结果
- 无法完成评分分桶
- 无法归档策略

## 三、系统性问题

### 3.1 无重试机制
**问题：**
- 策略生成失败后直接跳过，无重试
- LLM 偶发性错误（如网络超时、返回格式错误）无法自动恢复

**建议改进：**
```python
# 在 generate_strategies_for_symbol 中添加重试逻辑
MAX_RETRIES = 3
for attempt in range(MAX_RETRIES):
    try:
        yaml_path = await generator.generate_yaml_strategy(...)
        if yaml_path:
            break
    except Exception as e:
        if attempt == MAX_RETRIES - 1:
            logger.error(f"  ✗ 生成失败 ({strategy_name})，已重试 {MAX_RETRIES} 次: {e}")
        else:
            logger.warning(f"  ⚠️  生成失败 ({strategy_name})，重试 {attempt + 1}/{MAX_RETRIES}: {e}")
            await asyncio.sleep(5)
```

### 3.2 失败策略无追踪
**问题：**
- 失败策略只记录在日志中
- 无结构化的失败清单

- 在 `evidence_manager` 中添加 `save_failure_report` 方法
- 记录失败策略的完整上下文（prompt、错误、重试次数）
