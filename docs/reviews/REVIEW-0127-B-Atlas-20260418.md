### ⚠️ P4（新引入 Bug）— `_preprocess_yaml()` async/sync 混用

**修复意图**：调用 `ContractResolver` 做主力合约转换（上轮正确方向）  
**新 Bug 位置**：`tqsdk_backtest_client.py` 第 307-310 行

```python
loop = asyncio.get_event_loop()
main_contract = loop.run_until_complete(
    self._contract_resolver.get_main_contract(symbol)
)
```

`_preprocess_yaml()` 是同步方法，被 `async def submit_backtest()` 调用。在 async 上下文中，event loop 已在运行，调用 `loop.run_until_complete()` 会抛出：

```
RuntimeError: This event loop is already running
```

导致所有含 "0" 后缀合约的 YAML 提交 TqSdk 时必定抛异常（虽有 `except Exception` 兜底保持原值，但合约转换仍失败）。

**正确修复方案**：将 `_preprocess_yaml()` 改为 `async`，并在 `submit_backtest()` 中 `await` 调用：

```python
async def _preprocess_yaml(self, strategy: dict) -> dict:
    ...
    main_contract = await self._contract_resolver.get_main_contract(symbol)

# 调用处：
strategy = await self._preprocess_yaml(strategy)
```
